### Tests

- `operators::transform::tests::mesh` (new test-local helper): builds a
  4-vertex / 2-triangle `Tessellation` from caller-supplied positions so
  vertices can carry non-zero Z. The existing `quad()` helper has `z = 0`
  for every vertex, which would hide the Z axis under rotation/scale.
- `operators::transform::tests::rotation_y_90_deg_maps_xyz_to_z_y_neg_x`
  (new test): applies a +90° positive Y-axis rotation via quaternion
  `[0.0, sqrt(0.5), 0.0, sqrt(0.5)]`. Expected positions are hard-coded
  from the standard right-handed formula `(x, y, z) -> (z, y, -x)`,
  independent of `TransformOp::evaluate`. Input vertices
  `[1,0,0] [0,0,1] [2,3,-1] [-1,-2,4]` are asserted against
  `[0,0,-1] [1,0,0] [-1,3,-2] [4,-2,1]` within absolute tolerance `1e-5`.
- `operators::transform::tests::non_uniform_scale_multiplies_each_axis_independently`
  (new test): applies non-uniform scale `[2.0, 3.0, 4.0]` to vertices that
  all carry non-zero Z (`[1,1,1] [2,0,1] [0,2,3] [1,1,2]`). Expected
  positions are hard-coded from per-axis multiplication `(x*sx, y*sy, z*sz)`
  = `[2,3,4] [4,0,4] [0,6,12] [2,3,8]`, asserted within `1e-5`. All three
  scale axes are observably exercised.
- `operators::transform::tests::structural_hash_distinguishes_rotation_only_and_scale_only_changes`
  (new test): from a shared base transform, a rotation-only variant
  (only `rotation_quat_xyzw` differs) and a scale-only variant (only
  `scale` differs) each produce a hash distinct from the base hash, and
  distinct from each other. The existing `structural_hash_is_deterministic`
  test (translation-change coverage) is left unchanged.
