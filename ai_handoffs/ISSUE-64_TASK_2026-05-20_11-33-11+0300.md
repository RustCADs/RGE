## Deliverables

- Add `crates/editor-ui/tests/phase8_graph_diff_smoke.rs` as one focused
  integration test target.
- In that test, define a small test-local node payload such as `DiffNode {
  display_name: String, kind: String }` with `Clone`, `Debug`, `PartialEq`, and
  `Eq`.
- In that test, define a test-local helper that accepts `&dyn
  rge_kernel_graph_foundation::VizAdapter`, populates a temporary
  `rge_kernel_graph_foundation::Graph<DiffNode, String>` from `adapter.nodes()`
  and `adapter.edges()`, and returns `GraphSnapshot<DiffNode, String>`.
  Preserve each adapter `NodeId`, `EdgeId`, edge `src`, edge `dst`, node display
  name, node kind, and edge label exactly.
- Build an old and new `MaterialGraph` snapshot using real public
  `MaterialGraph` APIs. The old snapshot should contain `albedo -> output`
  with `MaterialEdge { src_port: PortType::Color, dst_port: PortType::Color }`;
  the new snapshot should add a `normal` node and a `normal -> output` edge
  with `MaterialEdge { src_port: PortType::Vector, dst_port:
  PortType::Texture }`.
- Build an old and new `AnimGraph` snapshot using real public `AnimGraph` APIs.
  The old snapshot should contain `idle -> run` with
  `AnimTransition::new("start_run")`; the new snapshot should add a `jump`
  state and a `run -> jump` transition with `AnimTransition::new("leap")`.
- Build an old and new `OperatorGraph` snapshot using real public cad-core
  APIs. The old snapshot should contain `Cuboid -> Transform`; the new snapshot
  should add one distinct second `Transform` node and one
  first-transform-to-second-transform edge on input port `0`.
- For each domain, call the same shared `GraphDiff::between(&old_snapshot,
  &new_snapshot)` API and assert exactly one added node and exactly one added
  edge, with the expected ids, projected node payloads, and projected
  `EdgeRecord<String>` data.
- Assert for each domain that `removed_nodes`, `removed_edges`,
  `changed_nodes`, and `changed_edges` are empty, `node_change_count() == 1`,
  `edge_change_count() == 1`, and `!diff.is_empty()`.

## Planner Notes

Use the existing Phase 8 editor-ui graph-domain smokes as reference material for
small public graph fixtures, but do not modify them. Unlike issue #63, this
task is not about `NodeGraphWidget`; it is specifically about proving one
graph-foundation `GraphDiff::between` smoke across all three domains from an
editor-ui integration test target.

The intended shape is a compact helper-driven test: build old snapshot, mutate
the same real graph by adding one node and one edge, build new snapshot, diff,
and assert the single addition for material, animation, and CAD operator graphs.
Snapshotting the public `VizAdapter` projection is deliberate: it keeps this
test outside domain-private internals while still exercising the shared
graph-foundation `GraphSnapshot` and `GraphDiff` path over stable ids and edge
records from each domain.
