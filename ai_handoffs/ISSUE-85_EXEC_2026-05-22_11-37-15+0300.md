    `Option<Instant>` pending mark, and an `Option<RecommendedWatcher>` so

    is `Option` so the `#[cfg(test)] for_test` constructor can skip it).
  - `GlbWatcher::new(PathBuf) -> notify::Result<Self>` builds the channel,

    glb's parent directory non-recursively. Construction failures
    (no-parent, notify backend error) bubble up.
  - `GlbWatcher::take_reload_request(&mut self, now: Instant) -> bool` is
    the only drain entry point. It non-blockingly `try_recv()`s every
    queued event, sets `pending_at = Some(now)` for each modify event whose
    `paths` contain the active source path, ignores notify-level errors,
    ignores non-modify events, ignores sibling-file modify events, and
    only fires once `now - pending_at >= debounce` (clearing the pending
    mark when it returns true). Producer-only ‚Äî no render-asset or shell
    field is touched here.
  - Pure helper `event_targets_path(&Event, &Path) -> bool` so the
    filter logic is unit-testable without a watcher instance.

  - New `struct EditorApp { shell: EditorShell, watcher: Option<GlbWatcher> }`

## Per-File Summary

The implementation strategy was Option 1 from the Planner Notes ‚Äî
wrap/delegate `EditorShell`'s `ApplicationHandler` from `rge-editor`.
This kept the `notify` dependency, the watcher logic, and the
debounce/filter state entirely inside the binary; **`crates/editor-shell`
required zero edits**, matching the smaller-source-surface preference.

The watcher is a strict request producer: `take_reload_request` returns
`bool` and never touches the shell. The wrapper's `window_event`
intercept calls `EditorShell::handle_asset_reload()` (the same entry
point the R-key path uses) for the actual reload semantics ‚Äî atomic
swap, PIE gate, warn-on-failure, prior-frame retention all remain inside
that single shell-side function.

`load_all_glb_meshes`, `reload_render_assets`, mesh / material / GPU
fields, and `prebuilt_render_*` slots are never accessed from the
watcher or the wrapper.

2. **Watcher construction failure policy**: when `GlbWatcher::new`
   returns `Err` (e.g. notify backend init failed, parent directory
   unreadable), the binary warn-logs and proceeds with `watcher = None`.
   This preserves manual R-key reload behaviour but silently disables
   automatic reload for the session. Acceptable for v0, or should this
   surface a louder signal (e.g. non-zero exit on watcher init failure)?

3. **Synthetic-event integration test posture**: the new composite test
   uses `GlbWatcher::for_test` (synthetic event injection) rather than
   driving the real notify backend against a real file. This avoids
   "unreasonable sleeps for notify/debounce timing" per the Halt
   Conditions ‚Äî debounce timing is deterministic via synthetic-`Instant`

   (`load_all_glb_meshes` from disk) and the **real** `handle_asset_reload`
   + `reload_render_assets` swap when GPU is available. Acceptable, or
   would a notify-backed end-to-end test be preferred even at the cost
   of sleep-based timing?
