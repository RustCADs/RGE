## Goal

Add the GitHub issue #39 cad-projection integration smoke proving that
`CadProjection::pick_face` can pick the top cap of a directly projected
square-profile `ExtrudeOp` root. The smoke must exercise the real projection
and picker path with a `BRepHandle` owner, then assert the returned
`FacePick` identifies the Extrude-bound entity, that owner, and the stable

## Deliverables

  `crates/cad-projection/tests/face_picking_smoke.rs` for a direct
  square-profile `ExtrudeOp` root.
- The test builds a deterministic square `Polygon2D` profile with four points,
  constructs `ExtrudeOp::new(profile, 1.0)`, adds it as
  `OperatorNode::Extrude`, sets that node as the graph root, commits the graph,
  and spawns the projected entity for that Extrude node.
- The spawned entity's `BRepHandle.brep_owner` is set to
  `Some(ENTITY_OWNER)` before `projection.tick`.
- The test casts a ray from above the center of the top cap, for example

  intersects the top cap interior rather than an edge or vertex.

  `entity` is the Extrude-bound entity, `owner` is `ENTITY_OWNER`, and
  `face_id` equals
  `BRepFaceId::for_extrude_face(ENTITY_OWNER, ExtrudeFaceTag::Top)`.
- The test should also assert a positive ray `t`; an exact `t` assertion is
  allowed if it is tied to the top cap at `z = 1.0`.

  result, deviations if any, and final git status.

## Planner Notes

`crates/cad-projection/tests/extrude_brep_face_id_lookup_smoke.rs` already
proves triangle-index lookup for Extrude. This dispatch is different: it must
exercise the ray picker in `face_picking_smoke.rs` and prove the picked top cap
flows through the same entity, owner, and stable Extrude Top-cap identity that
callers consume from `FacePick`.

The square profile should match the existing CCW unit-square convention used by
the Extrude projection smokes: `[0.0, 0.0]`, `[1.0, 0.0]`, `[1.0, 1.0]`,
`[0.0, 1.0]`, with extrusion length `1.0`. A ray through `[0.5, 0.5]` in XY is
inside both top-cap fan triangles and avoids edge or vertex ambiguity.
