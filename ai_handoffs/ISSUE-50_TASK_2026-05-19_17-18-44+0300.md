## Goal

Add a narrow AnimGraph RON snapshot round-trip smoke test that mirrors the existing MaterialGraph snapshot coverage. The dispatch should prove that a populated `AnimGraph` can round-trip its private graph substrate through `GraphSnapshot::to_ron`, `GraphSnapshot::from_ron`, and `GraphSnapshot::to_graph` without losing node payloads, edge ids, edge endpoints, or transition payloads.

## Deliverables

- Add the minimal serde support needed for `GraphSnapshot<AnimState, AnimTransition>` RON serialization, expected to be serde derives on `AnimState` and `AnimTransition` plus a `serde = { workspace = true }` dependency in `crates/anim-graph/Cargo.toml` if it is not already present.
- Add one unit test in `crates/anim-graph/src/lib.rs`'s existing `#[cfg(test)]` module, preferably named `snapshot_ron_round_trip_preserves_anim_payloads`.
- The test must build a populated `AnimGraph` with at least three states and at least two transitions.
- The test must capture the private substrate graph with `GraphSnapshot::from_graph(&g.graph)`, serialize it with `to_ron`, deserialize it with `from_ron`, restore it with `to_graph`, and assert the restored graph is equivalent to the original.
- The test must assert that node ids and `AnimState` payloads survive unchanged, edge ids and endpoints survive unchanged, and each `AnimTransition` trigger payload survives unchanged.

## Acceptance Criteria

- Only `crates/anim-graph/src/lib.rs` and, if needed, `crates/anim-graph/Cargo.toml` are changed for implementation.
- `AnimState` and `AnimTransition` can satisfy the serde bounds required by `GraphSnapshot::to_ron` and `GraphSnapshot::from_ron`.
- The smoke test uses the existing private `AnimGraph::graph` field from inside the crate's unit-test module; it must not add a public graph accessor solely for this test.
- The smoke test compares complete node pairs, such as `(NodeId, AnimState)`, and complete edge records, such as `(EdgeId, EdgeRecord<AnimTransition>)`, rather than checking counts only.
- The smoke test spot-checks the concrete transition ids returned by `add_transition` and verifies their source node, destination node, and `AnimTransition::trigger` payload after restore.
- No traversal, evaluation, blend-tree, runtime state-machine, editor, renderer, asset, ECS, `SnapshotParticipate`, architecture-lint, or cross-crate behavior is added.
- `cargo test -p rge-anim-graph` exits 0.
