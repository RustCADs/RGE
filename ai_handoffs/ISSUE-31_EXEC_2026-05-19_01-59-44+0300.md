### Tests

- `crates/cad-projection/tests/render_adapter_smoke.rs::round_fillet_render_mesh_face_labels_align_with_projection_lookup`:
  new additive `#[test]` covering `CadProjection::render_mesh_for` against a
  projected `CuboidOp -> RoundFilletOp` root. It:
  - constructs a deterministic `CuboidOp { 1.0, 1.0, 1.0 }`, takes real
    upstream edge IDs via `cuboid.brep_edge_ids(ENTITY_OWNER)`, and builds
    `RoundFilletOp::new(&cuboid, ENTITY_OWNER, vec![edges[0]], 0.1)` — no
    `BRepEdgeId` synthesized by hand;
  - commits a `CadGraph` with the Cuboid node feeding the RoundFillet node
    (`connect(cuboid_node, round_node, 0)`), RoundFillet node as root;
  - spawns the projected entity on the RoundFillet root node, sets
    `BRepHandle.brep_owner = Some(ENTITY_OWNER)`, then `tick`s;
  - calls both `projected_mesh(entity)` and `render_mesh_for(entity, &world)`
    on the RoundFillet-bound entity;
  - asserts `RenderMesh.face_labels` is `Some`, has one opaque `u64` per
    projected triangle (`render_labels.len() == projected.triangle_count()`),
    and is aligned by triangle index with the projected
    `TopologyFaceId` labels (`render_labels[i] == projected_labels[i].0`);
  - for every inherited non-degenerate renderer label, wraps the `u64` as
    `TopologyFaceId(label)`, resolves it through the upstream Cuboid face
    mapping `cuboid.brep_face_ids(ENTITY_OWNER)`, and asserts that exact
    `BRepFaceId` equals `brep_face_id_for_triangle(entity, tri, &world,
    graph.graph())` — not `Some(_)`, not "any face", not a sorted set;
  - asserts at least one inherited non-degenerate label is observed and at
    least one `TopologyFaceId::DEGENERATE.0` renderer label from
    RoundFillet-added cap/corner geometry remains present in the buffer.
- Updated the file-level module doc: added a fourth bullet for the
  RoundFillet renderer-label alignment invariant and an explicit test
  inventory listing all four `#[test]` functions.
