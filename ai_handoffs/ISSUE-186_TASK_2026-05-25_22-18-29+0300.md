## Goal

Persist the existing opt-in automation timing trace stream as JSONL events for
GitHub issue #186. `Invoke-AiDispatchAuto.ps1` and
`Invoke-AiDispatchQueue.ps1` already emit human-readable trace lines when
`-TraceTiming` or `RGE_AI_DISPATCH_TRACE_TIMING` enables tracing; this dispatch
must add a best-effort machine-readable JSONL companion without changing the
existing console trace text, dispatch behavior, publish policy, or retry
policy.

## Deliverables

- Extend the existing `Write-TimingTrace` helper in both
  `Invoke-AiDispatchAuto.ps1` and `Invoke-AiDispatchQueue.ps1` so every
  existing timing trace call still writes the same console line and also
  attempts to append one JSONL event when tracing is enabled.
- Keep the current tracing enablement contract exactly: JSONL persistence is
  active only when the existing `$script:TraceTimingEnabled` value is true,
  which is driven by `-TraceTiming` or
  `RGE_AI_DISPATCH_TRACE_TIMING=1|true|yes|on`.
- Preserve the exact existing console trace output format and message text:
  `[TRACE HH:mm:ss.fff +N.NNNs] $Message`. Do not rewrite existing
  `Write-TimingTrace` call-site messages.
- Write one compact JSON object per line. Each object must include these
  fields:
  - `timestamp`: event timestamp as an ISO 8601 string.
  - `elapsed_seconds`: elapsed seconds from the existing timing stopwatch as a
    JSON number, not a formatted string.
  - `script`: the script leaf name, either `Invoke-AiDispatchAuto.ps1` or
    `Invoke-AiDispatchQueue.ps1`.
  - `pid`: the current PowerShell process id as a JSON number.
  - `event`: the event name derived from the existing message without adding a
    new taxonomy. Use the substring before the first colon when present, such
    as `auto.tick` from `auto.tick: start (...)`; otherwise use the first
    whitespace-delimited token or the full message if no token exists.
  - `message`: the exact message string passed to `Write-TimingTrace`.
  - `repo`: repo context with at least `root` and `slug` values when available.
- Store trace files under `.ai/dispatch-trace/` unless the implementation uses
  another path that is already gitignored before this dispatch. The directory
  and files must be created only when tracing is enabled, and the trace
  filename must be process-specific enough to avoid concurrent append
  collisions, such as including the script name, timestamp, and `$PID`.
- Make all JSONL directory creation, serialization, and append operations
  best-effort. A trace write failure must not throw out of `Write-TimingTrace`,
  must not change the script exit code, and must not block dispatch progress.
- Keep the implementation compatible with Windows PowerShell 5.1.

## Acceptance Criteria

- The only source files changed are `Invoke-AiDispatchAuto.ps1` and
  `Invoke-AiDispatchQueue.ps1`.
- With tracing disabled, neither script creates a trace directory, creates a
  trace file, appends JSONL, or changes console output.
- With tracing enabled through `-TraceTiming`, both scripts append JSONL events
  for their existing timing trace calls.
- With tracing enabled only through `RGE_AI_DISPATCH_TRACE_TIMING`, both
  scripts append JSONL events for their existing timing trace calls. The auto
  driver must not need to pass `-TraceTiming` explicitly for the queue child to
  honor the inherited environment variable.
- Existing console trace lines remain byte-for-byte equivalent in shape and
  content to the current implementation for the same event messages. JSONL
  persistence may not add warnings, banners, summaries, or other console trace
  output.
- Each JSONL line is one compact JSON object on one line, parseable by
  `ConvertFrom-Json`, and carries `timestamp`, `elapsed_seconds`, `script`,
  `pid`, `event`, `message`, and `repo` fields.
- `elapsed_seconds` is numeric JSON. `pid` is numeric JSON. `message` preserves
  the exact message passed to `Write-TimingTrace`.
- Trace files are written only under an already ignored local scratch path,
  preferably `.ai/dispatch-trace/` because it is covered by the existing
  `/.ai/dispatch-*/` ignore rule. Trace files must not appear in the dispatch
  commit.
- If trace directory creation, serialization, or append fails, the script still
  emits the original console trace line and continues as though JSONL
  persistence were unavailable.
- The implementation does not add aggregation, alerting, dashboards, taxonomy
  labels, retry-policy changes, recovery routes, performance speedups,
  watchdog changes, preflight changes, docs, schemas, Rust changes, Cargo
  changes, tests, fixtures, dependencies, or new scripts.

## Planner Notes

Both target scripts already have the same trace-enablement pattern near the
top: `$script:TraceTimingEnabled`, `$script:TraceTimingStopwatch`, and
`Write-TimingTrace`. Keep the change local to that existing surface where
possible. A small helper that initializes a process-local JSONL path lazily and
another helper that appends compressed JSON inside a `try`/`catch` is likely
enough.

Use `ConvertTo-Json -Compress` or an equivalent PowerShell 5.1-compatible
serializer to produce exactly one compact JSON object per line. If nested repo
context is used, keep the conversion depth explicit. The `event` field is not
a new classification system; it is a mechanical extraction from the existing
message prefix so future tooling can group the same trace stream that humans
already read.

The safest trace location is `.ai/dispatch-trace/`, whose top-level directory
name matches the repository's existing `/.ai/dispatch-*/` ignore rule. Do not
choose `.ai/trace/` unless `git check-ignore` proves it is already ignored
without changing `.gitignore`.
