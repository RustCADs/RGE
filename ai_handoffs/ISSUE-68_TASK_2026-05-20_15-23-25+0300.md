## Goal

Implement GitHub issue #68 by adding the graph-viewer adapter surface for the
new `rge-script-graph` foundation wrapper. `ScriptGraph` already stores
data-only `ScriptNode` and `ScriptEdge` payloads on
`rge_kernel_graph_foundation::Graph`; this dispatch should expose that existing
structure through `rge_kernel_graph_foundation::VizAdapter` only, without adding
script execution, evaluator, runtime, editor UI, or script-host behavior.

## Deliverables

- `crates/script-graph/src/lib.rs` imports the graph-foundation view types and
  trait needed for the adapter: `EdgeView`, `NodeView`, and `VizAdapter`.
- `impl VizAdapter for ScriptGraph` exists in `crates/script-graph/src/lib.rs`.
- `VizAdapter::node_count` and `VizAdapter::edge_count` delegate directly to
  the private substrate graph counters and report the same values as the
  existing inherent `ScriptGraph::node_count` and `ScriptGraph::edge_count`.
- `VizAdapter::nodes` returns boxed borrowed `NodeView` items by iterating
  `self.graph.nodes()`. Each node view must expose:
  - `id`: the existing substrate `NodeId`;
  - `display_name`: `ScriptNode.key.as_str()`;
  - `kind`: the exact static string `ScriptNode`.
- `VizAdapter::edges` returns boxed borrowed `EdgeView` items by iterating
  `self.graph.edges()`. Each edge view must expose:
  - `id`: the existing substrate `EdgeId`;
  - `src`: the edge record source node id;
  - `dst`: the edge record destination node id;
  - `label`: `ScriptEdge.key.as_str()`.
- The adapter must be read-only and borrow existing graph payload data. Do not
  add side tables, cached view state, duplicate adjacency storage, graph
  traversal behavior, sorting layers, allocation-backed labels, or runtime
  semantics.
- Add focused unit tests in the existing `#[cfg(test)]` module for:
  - trait-observed counts and iterator counts through `&dyn VizAdapter`;
  - deterministic node views in substrate order, with ids, display names, and
    `ScriptNode` kind asserted;
  - deterministic edge views in substrate order, with ids, endpoints, and
    labels asserted.
- Preserve the existing `ScriptGraph`, `ScriptNode`, `ScriptEdge`,
  `ScriptGraphError`, snapshot round-trip, duplicate detection, dangling
  endpoint, and count behavior.

## Acceptance Criteria

- `crates/script-graph/src/lib.rs` is the only implementation/test file
  changed.
- No new implementation, test, doc, config, schema, script, generated,
  manifest, lockfile, or architecture-lint file is added.
- `ScriptGraph` can be used as `&dyn VizAdapter`.
- Node views expose exactly one view per existing `ScriptNode`, in the
  deterministic order yielded by the inner graph. The tests should derive the
  expected order from the returned `NodeId`s rather than assuming insertion
  order.
- Edge views expose exactly one view per existing `ScriptEdge`, in the
  deterministic order yielded by the inner graph. The tests should derive the
  expected order from the returned `EdgeId`s rather than assuming insertion
  order.
- Node view `display_name` values are borrowed from `ScriptNode.key`, and node
  view `kind` is exactly `ScriptNode`.
- Edge view `label` values are borrowed from `ScriptEdge.key`, and edge view
  `src` and `dst` match the stored edge records.
- The crate remains data-only: no function compiles, loads, executes,
  interprets, schedules, traverses, evaluates, hosts, or edits scripts.
- `cargo test -p rge-script-graph` exits 0.

## Planner Notes

Issue #67 established `rge-script-graph` as a data-only graph-foundation
wrapper and intentionally left graph-viewer adapter work out of that first
slice. Issue #68 now authorizes only that adapter surface.

Use `crates/anim-graph/src/lib.rs` and `crates/material-graph/src/lib.rs` as
local reference patterns for the `VizAdapter` implementation and tests, but do
not copy unrelated invalidation, diff, runtime, editor, or integration
coverage. The deterministic order should come from `Graph`'s existing ordered
iterators; the adapter itself should not sort or duplicate storage.
