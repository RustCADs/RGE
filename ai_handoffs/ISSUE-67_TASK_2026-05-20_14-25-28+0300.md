## Deliverables

- `crates/script-graph/src/lib.rs` declares `//! Failure class:
  snapshot-recoverable` near the top of the crate docs and keeps
  `#![forbid(unsafe_code)]`.
- `crates/script-graph/src/lib.rs` exposes a minimal public data model:
  `ScriptGraph`, `ScriptNode`, `ScriptEdge`, and `ScriptGraphError`.
- `ScriptGraph` privately wraps `rge_kernel_graph_foundation::Graph` with
  `ScriptNode` node payloads and `ScriptEdge` edge payloads. Do not add
  parallel adjacency storage, runtime state, evaluator state, or script-host
  handles.
- `ScriptNode` is an uninterpreted data-only node payload, preferably a single
  public `key: String` with a small constructor. The `NodeId` for insertion is
  content-derived from that key's bytes.
- `ScriptEdge` is an uninterpreted data-only edge payload, preferably a single
  public `key: String` with a small constructor. The `EdgeId` for insertion is
  content-derived from the source id, destination id, and edge key bytes so
  identical links are rejected while different edge keys between the same nodes
  remain distinct.
- `ScriptGraph` provides only the minimal mutation and count surface:
  `new`, `Default`, `add_node`, `connect`, `node_count`, and `edge_count`.
  Additional read-only helpers are allowed only if needed by tests and still
  remain data-only.
- `ScriptGraphError` is a thin wrapper over
  `rge_kernel_graph_foundation::GraphError` with `Debug`, `Display`,
  `std::error::Error`, `PartialEq`, `Eq`, and `From<GraphError>` behavior.
- `crates/script-graph/Cargo.toml` depends only on existing workspace
  dependencies needed for this data wrapper, expected to be
  `rge-kernel-graph-foundation = { workspace = true }` and
  `serde = { workspace = true }`.
- `tools/architecture-lints/exemptions.toml` removes only the existing
  `failure-class` exemption entry whose file is
  `crates/script-graph/Cargo.toml`.
- Unit tests in `crates/script-graph/src/lib.rs` cover stable node ids,
  duplicate node rejection, successful edge insertion and counts, duplicate
  edge rejection, same endpoints with different edge keys, dangling endpoint
  rejection, default empty counts, and a `GraphSnapshot` RON round trip over
  the private substrate graph.
