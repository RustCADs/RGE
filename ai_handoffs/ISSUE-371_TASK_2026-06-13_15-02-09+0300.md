## Goal

Add the smallest viewport-only right-button camera orbit slice in
`editor-shell`: pressing the right mouse button over the current Viewport tab
body starts an orbit drag, cursor movement rotates `EditorCameraState.eye`
around its existing target, and releasing the right mouse button stops the drag.
This follows task 115's viewport-only mouse-wheel zoom and must stay inside the
existing camera state, cursor tracking, and viewport hit-test boundary.

## Deliverables

- Camera orbit math implemented on `EditorCameraState` or a shell-private helper
  using only finite, target-preserving, distance-preserving eye rotation.
- `EditorShell` lifecycle state for an active right-button viewport orbit drag,
  including the last cursor position needed to convert cursor deltas into orbit
  deltas.
- `WindowEvent` routing that starts orbit only on right-button press over the
  Viewport tab body, applies orbit during cursor movement while active, and
  stops orbit on right-button release.
- Focused unit tests for orbit start gating, cursor-delta orbit behavior,
  release stop behavior, and no-op cases.
- Normal ISSUE-371 bookkeeping only if needed: task-list/status/history updates
  and the executor's own ISSUE-371 handoff/log artifacts.

## Acceptance Criteria

- Right-button press over the Viewport starts orbit only when a cursor position
  exists and the existing viewport hit-test reports true.
- Cursor movement while orbit is active rotates `editor_camera.eye` around
  `editor_camera.target`; the target, eye-target distance, up vector, FOV, near
  clip, far clip, and finite camera state are preserved.
- Right-button release stops orbit. Cursor movement after release does not keep
  rotating the camera.
- Right-button press with no cursor, no egui host, no viewport rect, or a
  non-viewport hit-test result is a no-op and must not panic.
- Drag/cursor movement with no active orbit is a no-op.
- Left-click face picking, viewport mouse-wheel zoom, View menu Reset/Zoom,
  command-palette/menu routing, and keyboard accelerator routing continue to
  pass their existing tests.
- No commands, accelerators, menu items, host/UI edits, Cargo changes, plugin
  runtime/discovery/loading, clipboard behavior, CAD graph/projection mutation,
  undo/dirty policy, pan, frame/focus, pointer-capture/window-grab policy,
  camera persistence, or generalized input routing are added.
