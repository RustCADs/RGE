## Goal

Add visual acceptance coverage for the M3 imported-normal path by committing a smooth-normal glTF fixture and a headless pixel/readback test that renders imported normals against the same geometry rendered with flat recomputed normals. The coverage must prove that glTF `NORMAL` accessors create a driver-noise-resistant visible difference, not just that normal buffers are non-empty.

## Deliverables

- A committed binary fixture at exactly `crates/io-gltf/tests/fixtures/smooth_normal_sphere.glb`.
- The fixture must contain a sphere-like indexed glTF mesh with per-vertex normals that differ meaningfully from per-triangle flat normals. A low-poly icosphere or latitude-longitude sphere is acceptable if its normals are normalized vertex/surface normals and the visible flat-vs-smooth difference is large.
- One headless pixel/readback test in `rge-editor` tests, added inside `editor/rge-editor/src/main.rs`, that:
  - loads `smooth_normal_sphere.glb` through the real `load_all_glb_meshes` path for the imported-normal render;
  - builds a same-geometry flat-recompute render in test code by passing `None` for normals to `RenderMesh::from_buffers_with_attributes` or an equivalent test-local construction;
  - renders both variants through `rge_editor_shell::visual_test_harness::render_one_frame_to_readback`;
  - scans the central horizontal row and asserts a committed numeric distribution threshold between imported-normal and flat-recomputed pixels.
- An execution report packet describing changed files, the exact central-row metric and threshold, commands run, and whether the local visual test executed or skipped due to no headless GPU adapter.

## Acceptance Criteria

- `crates/io-gltf/tests/fixtures/smooth_normal_sphere.glb` exists at that exact path after the dispatch and is not merely generated transiently in a temp directory.
- Importing the fixture yields one drawable mesh with non-empty normals aligned 1:1 with positions before render-mesh vertex tripling.
- The visual test compares two render readbacks from the same fixture geometry:
  - imported M3 normals;
  - `None` normals, causing flat recompute.
- The test includes a measurable pixel/readback assertion over the central row. It must not rely on a visual-only claim, a screenshot, a log line, a single center pixel, or an in-memory normal-buffer assertion alone.
- The final assertion must use a numeric literal threshold selected from measured output and committed in the test. Acceptable metric shapes include central-row RGB-sum delta, mean absolute luma/RGB delta, adjacent-difference distribution, or variance distribution, but the threshold must have an absolute floor large enough to reject driver noise. As a rule of thumb, use at least `mean_abs_rgb_delta > 8.0` or `sum_abs_rgb_delta > 300` over common on-mesh central-row pixels, or document an equally strong literal threshold in the test failure message.
- The test must assert that both renders hit a meaningful number of on-mesh central-row pixels, for example more than 8, before comparing distributions.
- No production behavior changes are allowed. The diff must remain fixture/test coverage only.

## Planner Notes

`rge-editor` is the preferred test location because its inline tests can reach the private `load_all_glb_meshes`, `bake_positions`, and render-shell helpers while already depending on `rge-io-gltf`, `rge-brep-render`, `rge-editor-shell`, `rge-gfx`, and `wgpu` in the existing shape. Keep the comparison test-local: load/import the fixture, render the existing imported-normal path, then construct the flat-recompute counterpart from the same mesh data without altering production loader code.

The existing comments in `load_all_glb_meshes_preserves_input_normals_when_present` already identify the gap: cube normals are present but match flat recompute, so they do not visually distinguish imported normals from recomputed normals. This dispatch closes only that coverage gap.
