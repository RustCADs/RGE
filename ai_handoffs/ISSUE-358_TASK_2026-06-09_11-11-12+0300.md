## Deliverables

- A host-local shortcut-help open/close affordance in `EguiHost` that can display an egui window without enqueuing a `Command`.
- A pure projection from `ProjectedMainMenu` to shortcut-help rows. Each row must carry menu group, row label, optional shortcut display, command diagnostic id, and enabled state.
- Shortcut-help rows for File, Edit, Play, View, and Plugins that preserve the existing `ProjectedMainMenu` group order and each group's entry order.
- Unit tests covering shortcut-help row projection, executable shortcut rows, passive shortcut hint rows, disabled-state preservation, plugin rows, and non-interference with command-palette/menu-command behavior.
- Any normal ISSUE-358 execution report or status bookkeeping required by the handoff workflow.

## Acceptance Criteria

- The shortcut-help row projection accepts a `ProjectedMainMenu` reference as its input and does not resolve menus, inspect `MenuRegistry`, inspect editor-ui menu definitions, call plugin discovery/runtime/loading, or alter accelerator routing.
- The row order is exactly File entries, then Edit entries, then Play entries, then View entries, then Plugins entries, with each group preserving the order already present in the corresponding `ProjectedMainMenu` vector.
- Rows include both executable shortcut displays and passive shortcut hints already present in `ProjectedMainMenu`. Entries with no shortcut still appear with an empty or absent shortcut field.
- Every row shows or exposes the menu group, label, shortcut display, `Command::diagnostic_id()` value, and current enabled state.
- Plugin rows are included only from `ProjectedMainMenu.plugins`; the implementation must not load, discover, query, register, or execute plugins for the help window.
- Opening, closing, or interacting with shortcut help does not push to `menu_command_handoff`, does not mutate command-palette recents or pins, does not alter menu enablement, does not close the command palette, and does not change accelerator execution.
- The existing menu bar behavior remains unchanged for File/Edit/Play/View/Plugins entries and shortcut-conflict diagnostics.
- Focused tests named or filtered by `shortcut` cover the shortcut-help behavior and can be run with the focused gate below.
