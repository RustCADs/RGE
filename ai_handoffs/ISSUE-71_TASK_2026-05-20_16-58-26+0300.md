## Goal

Add one focused ScriptGraph unit-test smoke proving graph-foundation invalidation can propagate through the script wrapper by deriving downstream dependents from the wrapper's existing graph substrate, specifically from outgoing edge ids and each edge record's `dst`. The test should cover diamond convergence dedup so the shared downstream script node is invalidated exactly once, mirroring the existing MaterialGraph and AnimGraph invalidation smoke style without changing production API.

## Deliverables

- Add exactly one new unit test named `invalidation_propagates_through_script_outgoing_edges` in `crates/script-graph/src/lib.rs`'s existing test module.
- The test must construct a ScriptGraph diamond, for example `root -> left`, `root -> right`, `left -> join`, and `right -> join`.
- The test must register an `InvalidationListener` recorder, call `Invalidation::mark_dirty`, and assert that the dirty root plus all downstream script nodes are delivered exactly once.
- The dependents closure passed to `mark_dirty` must derive children from `g.graph.outgoing(node)` and `g.graph.edge(eid).map(|record| record.dst)`, proving no separate side table or production API is needed.
- The expected listener order must account for substrate outgoing-edge ordering by deriving the first and second child from the concrete `EdgeId`s returned by the two root outgoing edges, as the MaterialGraph/AnimGraph smoke tests do.
- The convergence node must have an explicit assertion proving it appears exactly once.
