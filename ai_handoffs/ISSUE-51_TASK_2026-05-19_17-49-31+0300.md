## Deliverables

- Add exactly one new unit test in `crates/material-graph/src/lib.rs`'s existing `#[cfg(test)] mod tests`; a clear name such as `structural_diff_reports_added_material_node_and_edge` is preferred.
- The test must build an initial `MaterialGraph`, insert at least two nodes and one edge, and take an old snapshot with `GraphSnapshot::from_graph(&g.graph)`.
- The test must mutate the same graph after the old snapshot by adding exactly one additional material node and exactly one additional material edge connected to that new node.
- The test must take a new snapshot with `GraphSnapshot::from_graph(&g.graph)`, run `GraphDiff::between(&old_snapshot, &new_snapshot)`, and assert the structural diff results.
- The test must assert that `diff.added_nodes` contains exactly the newly added `NodeId` with the expected `MaterialNode` payload/key.
- The test must assert that `diff.added_edges` contains exactly the newly added `EdgeId` with the expected `EdgeRecord`: source node, destination node, and `MaterialEdge` port payload.
- The test must assert that no nodes or edges were removed and no existing node or edge records were changed.
