- `crates/material-graph/src/lib.rs::tests::structural_diff_reports_added_material_node_and_edge`:
  new unit test pinning that `MaterialGraph` participates in the
  graph-foundation structural-diff path. It builds an initial graph
  (`albedo`, `output` nodes joined by a `Color->Color` edge), takes an old
  snapshot via `GraphSnapshot::from_graph(&g.graph)`, then mutates the same
  graph by adding exactly one node (`normal`) and exactly one edge
  (`normal -> output`, `Vector->Texture`). It takes a new snapshot, runs
  `GraphDiff::between(&old_snapshot, &new_snapshot)`, and asserts:
  - `added_nodes` has len 1 and `added_nodes[&normal]` equals
    `MaterialNode { key: "normal" }` (concrete id + payload, not a count).
  - `added_edges` has len 1 and `added_edges[&new_edge_id]` equals
    `EdgeRecord { src: normal, dst: output, data: MaterialEdge { Vector, Texture } }`
    (concrete id + full record, not a count).
  - `removed_nodes`, `removed_edges`, `changed_nodes`, and `changed_edges`
    are all empty.
- Test-module import line extended from
  `{EdgeRecord, GraphError, GraphSnapshot}` to
  `{EdgeRecord, GraphDiff, GraphError, GraphSnapshot}` so the new test can
  name `GraphDiff`.
