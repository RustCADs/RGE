`cad_entity`, `cad_world`, `BRepHandle`, and `CadProjection::render_mesh_for`
state only. This follows task 154 in `.ai/dispatch.tasks.md` and must leave the
autonomous loop re-armed with exactly one bounded task 155 audit, unless a

## Acceptance Criteria

- `CadSceneInspection` reports tracked-CAD entity presence separately from
  tracked-CAD entity liveness and tracked-CAD render-mesh availability.
- `EditorShell::cad_scene_inspection(&self)` remains a pure read. It must not
  tick projection, spawn/despawn entities, mutate CAD graph/projection/world,
  mutate selection, mutate camera, enqueue actions, initialize render state,
  upload meshes, touch GPU resources, mark dirty, save, load, open, or close.
- Fresh-shell inspection reports no tracked entity, no live tracked entity, and
  no tracked render mesh.
- First-cuboid-add inspection reports the stored tracked id is present, live,
  and renderable; existing graph/world/projection, node/root, `BRepHandle`,
  `render_mesh_for`, prebuilt-render, uploaded-mesh, and frameability assertions
  remain covered.
- A rejected non-empty add leaves the inspection snapshot unchanged.
- The restore/despawn cleanup test proves `tracked_cad_entity_present == true`
  while the new liveness/render-mesh fields report false, with no live
  `BRepHandle` entity and no stale `render_mesh_for` mesh.
- No menu item, shortcut, command-palette entry, UI/host route, plugin command,
  CommandBus `Action`, global undo/redo, dirty/save-mark, save/load/open/close
  behavior, camera behavior, viewport behavior, render-path refresh, CAD-core
  behavior, or cad-projection behavior is added or changed.
- Exactly one bounded next-task-source audit task 155 is appended per the
  Self-rearm protocol, carrying its own `MAY edit`, `MUST NOT edit`,
  `Done criteria`, `Verification`, `Halt conditions`, dispatcher-snapshot
  evidence rule, current-source comparison requirement, and copied Self-rearm
  final-step requirement; or one `NEEDS_HUMAN_RECORDED:` line is appended
  instead. No other task is added.
