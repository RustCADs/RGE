## What I Changed

### Source

- `crates/editor-egui-host/src/palette_recent.rs`: Added a host-local persistence helper that stores only newline-delimited `Command::diagnostic_id()` strings, validates one-line ids, caps/deduplicates most-recent-first ids, creates parent directories on save, and provides a nonfatal load-or-empty wrapper.
- `crates/editor-egui-host/src/lib.rs`: Added the `palette_recent` module, a `command_palette_recent_path` field, constructor loading from the deterministic default config path, and a private palette activation helper that records, attempts to save, logs save failures at debug level, and enqueues through `MenuCommandHandoff`.
- `crates/editor-egui-host/src/lib.rs`: Kept main-menu activation branches unchanged as direct `menu_commands.push(cmd.clone())` calls; only the command-palette activation branch calls the recent-record/save helper.

### Tests

- `crates/editor-egui-host/src/menu_tests.rs::command_palette_recent_persistence_round_trips_ids_only`: Verifies the persisted file contains only diagnostic id lines and loads back in order.
- `crates/editor-egui-host/src/menu_tests.rs::command_palette_recent_persistence_caps_and_deduplicates_ids`: Verifies persistence normalization preserves first-seen most-recent order, removes duplicates, and enforces the existing cap.
- `crates/editor-egui-host/src/menu_tests.rs::command_palette_recent_missing_unreadable_or_corrupt_persistence_is_nonfatal`: Verifies missing, directory/unreadable, and corrupt recent files fall back to an empty list at the host boundary.
- `crates/editor-egui-host/src/menu_tests.rs::command_palette_recent_persisted_stale_and_disabled_ids_are_not_promoted`: Verifies loaded stale ids are ignored and disabled persisted ids remain in the original-order remainder.
- `crates/editor-egui-host/src/menu_tests.rs::command_palette_recent_palette_activation_records_saves_and_enqueues`: Verifies successful palette activation records, saves, and still enqueues through `MenuCommandHandoff`.
- `crates/editor-egui-host/src/menu_tests.rs::command_palette_recent_unwritable_save_does_not_block_palette_dispatch`: Verifies save failure at an unwritable path is nonfatal for in-memory recents and command dispatch.
- `crates/editor-egui-host/src/menu_tests.rs::command_palette_recent_main_menu_activation_does_not_record_or_persist`: Verifies a main-menu style handoff push does not mutate or persist command-palette recents.
