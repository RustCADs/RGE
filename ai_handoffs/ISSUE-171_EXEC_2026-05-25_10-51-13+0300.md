- `crates/rge-scene-loader/src/lib.rs`:
  - Public entry point
    `pub fn load_scene_into_world(scene: &rge_data::Scene) -> Result<World, SceneLoadError>`.
  - `SceneLoadError` enum with `UnsupportedComponent` and `Deserialize`
    variants (typed source: `ron::de::SpannedError`).
  - Spawn loop walks every `Scene::entities` first, converting each
    `rge_data::EntityId` via `rge_kernel_ecs::EntityId::from_ulid(*scene_id.as_ulid())`
    and calling `World::spawn_with_id(...)` (never `World::spawn()`).
  - Component loop dispatches through a literal four-arm match on
    `component.type_id.as_str()` for exactly
    `"rge::components::Transform"`, `"rge::components::Camera"`,
    `"rge::components::Light"`, `"rge::components::Visibility"`. Each arm
    calls the matching `ron::from_str::<T>(...)` and `World::insert(ecs_id, value)`
    with the original component crate types — no wrappers, no erased path,
    no reflection. Any other `type_id` returns
    `SceneLoadError::UnsupportedComponent`; a RON parse failure returns
    `SceneLoadError::Deserialize`.
  - Scene relations and `root_entities` are intentionally not consumed
    (hierarchy materialization is out-of-scope per the Constraints).
- `Cargo.toml` (root): single member-list insertion of
  `"crates/rge-scene-loader"` adjacent to `"crates/rge-data"`. No

  or other metadata changes.
- `Cargo.lock`: one new `[[package]] name = "rge-scene-loader"` entry
