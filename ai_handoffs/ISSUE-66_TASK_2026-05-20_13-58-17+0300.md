## Goal

Add the missing `crates/io-3mf` workspace member for GitHub issue #66. PLAN
sections 1.6.5 and 13 list `io-3mf` as a required import/export authority, and
the current handoff/status docs identify that crate as entirely missing. This
dispatch should close only that workspace-shape gap by adding a stub crate that
matches the existing unimplemented importer stubs; it must not implement 3MF
parsing, exporting, dependencies, or any behavior.

## Planner Notes

Use `crates/io-obj`, `crates/io-stl`, `crates/io-step`, and `crates/io-audio`
as the local stub convention. Those crates have simple manifests, empty
dependency tables, workspace lints, and crate-level docs that say the
architecture is frozen with implementation pending. Because the current
`failure-class` lint checks Tier-2 crate manifests, the new dependency-free
stub will likely need the same rollout-debt exemption style as those adjacent
importer stubs unless the executor chooses a protocol-safe equivalent that
still matches the convention and passes the gate.

This dispatch is intentionally narrow. It closes the PLAN-listed workspace
member gap called out in the handoff/status docs; a future issue should own all
actual 3MF format design and implementation.
