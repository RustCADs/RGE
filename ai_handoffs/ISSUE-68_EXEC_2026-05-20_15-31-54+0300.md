- `crates/script-graph/src/lib.rs` (`impl VizAdapter for ScriptGraph`):
  Inserted a new "Graph-viewer adapter" section immediately after
  `script_edge_id`. The impl is purely a borrowing view surface:
  - `node_count` / `edge_count` delegate straight to
    `self.graph.node_count()` / `self.graph.edge_count()`.
  - `nodes()` iterates `self.graph.nodes()` and constructs a `NodeView` per
    pair with `id`, `display_name: node.key.as_str()`, and the static
    `kind: "ScriptNode"`.
  - `edges()` iterates `self.graph.edges()` and constructs an `EdgeView`
    per pair with `id`, `src`, `dst`, and `label: record.data.key.as_str()`.
  No new fields, side tables, cached views, traversal, sorting, allocation-
  backed labels, or runtime behavior were introduced — the adapter borrows
  existing substrate state only.

### Tests

Added three focused tests inside the existing `#[cfg(test)] mod tests` block,
placed just before the existing `snapshot_ron_round_trip_preserves_script_payloads`
test (so the snapshot test remains last):

- `tests::viz_adapter_counts_observed_through_trait` — builds three nodes
  + two edges, then checks via `&dyn VizAdapter` that
  `adapter.node_count() == 3`, `adapter.edge_count() == 2`,
  `adapter.nodes().count() == adapter.node_count()`, and
  `adapter.edges().count() == adapter.edge_count()`.
- `tests::viz_adapter_node_views_match_substrate_order` — builds three
  `ScriptGraph` nodes (`entry`, `body`, `exit`), then derives the expected
  order by sorting the returned `NodeId`s (per the acceptance criterion
  that the test must not assume insertion order). For each returned
  `NodeView`, asserts `id`, `display_name`, and that `kind == "ScriptNode"`.
- `tests::viz_adapter_edge_views_match_substrate_order` — builds three
  nodes and two edges (`flow`, `done`), derives the expected order by
  sorting the returned `EdgeId`s, then asserts `id`, `src`, `dst`, and
  `label` on each returned `EdgeView`.
