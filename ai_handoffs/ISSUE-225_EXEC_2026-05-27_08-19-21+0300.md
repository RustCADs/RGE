  conventions already established by the sibling `runtime/runtime-headless`

  the existing list shape). No version, checksum, source URL, or other
  package node was touched; the three named packages were already present
  elsewhere in the lockfile.
- `editor/rge-editor/src/main.rs`:
  - Added `scene_path: Option<PathBuf>` to the private `Cli` struct (alongside
    the existing `glb_path` field).
  - Added `CliError::MissingScenePath` and `CliError::GlbAndSceneConflict`
    variants, with `Display` impls that match the one-line stderr style of
    the existing `MissingGlbPath` / `UnknownArg` variants.
  - Extended `parse_args` with a `--scene <path>` arm. Each of the two load
    flags rejects the other when already set, producing
    `CliError::GlbAndSceneConflict` regardless of flag order.
  - Added a private `SceneLoadCliError` enum + Display impl and a pure
    `load_scene_world_from_path` helper. The helper dispatches on the file
    name (`Path::extension()` returns `None` for the canonical `.rge-project`
    name because Rust treats a leading-dot-only filename as having no
    extension): for a literal `.rge-project` it parses `rge_data::Project`,
    resolves the first scene relative to the project directory, parses
    `rge_data::Scene`, then calls `rge_scene_loader::load_scene_into_world`.
    For a `*.rge-scene` filename it parses the file directly as
    `rge_data::Scene` and calls the loader. The helper qualifies
    `rge_data::Scene` and `rge_data::Project` at the use site to avoid a name
    collision with the already-imported `rge_io_gltf::Scene` (per the planner
    notes).
  - Extended `main()` with a `--scene` branch placed before the existing
    `--glb` branch. The branch calls the helper, wraps the resulting
    `rge_kernel_ecs::World` into a fresh `rge_editor_shell::world::World` by
    overwriting the wrapper's inner kernel state via `kernel_mut()`, hands
    it to `EditorShell::with_world(shell_world)` (single call site, no CAD
    projection, no `BRepHandle`, no `with_world_projection_graph`), runs
    the same winit event loop with `EditorApp { shell, watcher: None }`, and
    returns. The existing `--glb` and default cuboid branches are unchanged
    except for the one-line usage banner now mentioning `--scene`.

One implementation note that is *consistent with* the packet but worth
flagging for the reviewer: `EditorShell::with_world` takes the
`rge_editor_shell::world::World` *wrapper*, not the
`rge_kernel_ecs::World` the loader returns. The wrapper's only public
ingress for an externally-built kernel world is `kernel_mut()`, so the
`--scene` branch constructs an empty wrapper via
`rge_editor_shell::world::World::new()`, overwrites its inner kernel
via `*shell_world.kernel_mut() = world;`, then passes the wrapper to
`EditorShell::with_world(shell_world)`. This preserves the wrapper's
TimeScale-resource contract (`with_world` installs the default when the
caller has not pre-inserted one â€” the loader never does). The
constructor call is `EditorShell::with_world(shell_world)`, satisfying
the acceptance criterion that the `--scene` branch use `with_world`
(not `with_world_projection_graph`).
