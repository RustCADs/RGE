## Deliverables

- `Command::Delete` in `EditorShell::route_menu_command` chooses the CAD delete
  route only for exact single selection equal to `self.cad_entity`.
- Non-CAD, mixed, and empty selections keep the existing wrapper-world
  `delete_selected_entities` behavior; `Command::Cut` remains wrapper-world only.
- Stale/invalid selected CAD preflight failures from `delete_current_cad_cuboid`
  are logged and swallowed without fallback, selection clearing, or bus-stack
  growth.
- Focused lifecycle tests cover the CAD route, undo/redo, stale render/selection
  cleanup, dirty/save cursor behavior, non-CAD/mixed/empty/stale cases, and
  existing wrapper Delete/Cut behavior.
- `.ai/dispatch.tasks.md` has exactly one appended bounded task 165 gated audit
  block carrying the required self-rearm instruction verbatim; task 164 does not
  directly record a `NEEDS_HUMAN_RECORDED` marker unless appending task 165 is
  impossible and the executor halts with that reason.

## Acceptance Criteria

- The route predicate is a direct ID comparison: selection length is exactly
  one, `self.cad_entity` is `Some(e)`, and the sole selected entity is `e`.
- Empty selection does not delete a CAD cuboid just because one exists.
- A single non-CAD selection and a mixed CAD plus non-CAD selection use the
  existing wrapper-world Delete path and do not mutate CAD graph/projection/world
  state.
- Exact selected tracked CAD uses `delete_current_cad_cuboid`, advances the
  CommandBus cursor/dirty state, clears `cad_entity`, despawns the tracked B-Rep
  entity, restores the empty parent checkpoint, prunes entity/face selections for
  the deleted CAD entity, and leaves no stale `render_mesh_for` lookup.
- Undo/redo after routed CAD Delete restores a fresh tracked entity and then
  re-deletes it without resurrecting the old entity id or leaving stale
  selection/projection state.
- Exact selected stale/invalid CAD logs a `tracing::warn!` under
  `target: "rge::editor-shell::menu"` and swallows the error without falling
  back to `delete_selected_entities`, without clearing selection, and without
  growing or moving the bus stack.
- `Command::Cut`, `cut_selected_entities`, `delete_current_cad_cuboid`,
  `DeleteCurrentCadCuboid`, `ShellActionContext`, the `Action` contract, and
  `lifecycle/mod.rs` public exports are unchanged.
- The focused filters `route_menu_command`, `delete_selected`, and
  `cut_selected` must run meaningful tests. Add or rename tests within
  `crates/editor-shell/src/lifecycle/tests.rs` if necessary so these filters do
  not silently report zero matching tests.
- `crates/editor-actions` remains generic and contains no CAD/editor-shell
  references per the required grep.
