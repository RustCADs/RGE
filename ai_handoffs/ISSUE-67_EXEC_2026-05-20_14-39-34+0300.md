## What I Changed

### Source

- `crates/script-graph/src/lib.rs`: full rewrite of the stub. Declares
  `//! Failure class: snapshot-recoverable` near the crate doc-comment
  block, keeps `#![forbid(unsafe_code)]`, and exposes the minimal public
  data model `ScriptGraph` / `ScriptNode` / `ScriptEdge` /
  `ScriptGraphError`. `ScriptGraph` privately wraps
  `Graph<ScriptNode, ScriptEdge>`; `ScriptNode { pub key: String }` and
  `ScriptEdge { pub key: String }` are uninterpreted data-only payloads
  with `::new(&str)` constructors. `add_node` derives the `NodeId` from
  the key bytes via `NodeId::from_bytes`. `connect` derives the `EdgeId`
  from the source id, destination id, and edge key bytes via
  `script_edge_id`, mirroring the anim-graph pattern so identical edges
  collide while differing edge keys between the same endpoints stay
  distinct. The public surface is exactly `new`, `Default`, `add_node`,
  `connect`, `node_count`, and `edge_count` — no extra read helpers, no
  `VizAdapter` impl, no traversal, no evaluator surface.
- `ScriptGraphError`: thin newtype over `GraphError` carrying the inner
  failure as a `pub` field, with `Debug`, `Display`, `std::error::Error`
  (delegating `source` to the wrapped `GraphError`), `PartialEq`, `Eq`,
  and `From<GraphError>` — matching the spec.
