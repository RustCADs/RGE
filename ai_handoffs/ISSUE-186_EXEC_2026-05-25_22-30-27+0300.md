### Source

- `Invoke-AiDispatchAuto.ps1` (around the existing trace block, previously lines 84-93): introduced four new script-scoped state variables (`TraceTimingScriptLeaf`, `TraceTimingJsonlPath`, `TraceTimingJsonlInitialized`, and a leaf-name constant), added an `Initialize-TimingTraceJsonl` helper that lazily creates `.ai/dispatch-trace/<leaf>-<timestamp>-<pid>.jsonl` only when tracing is enabled, and extended `Write-TimingTrace` to emit the original `Write-Output` console line first, then attempt one compact JSONL append inside a `try`/`catch` so write failures are swallowed.
- `Invoke-AiDispatchQueue.ps1` (same trace block, previously lines 66-75): identical extension with the script leaf set to `Invoke-AiDispatchQueue.ps1`, so the child queue process emits its own JSONL stream when `Invoke-AiDispatchAuto.ps1` runs it under `RGE_AI_DISPATCH_TRACE_TIMING` (the env var is inherited automatically; no `-TraceTiming` plumbing change was needed in the auto driver).

JSONL contract per line:
- One compact JSON object, `ConvertTo-Json -Compress -Depth 5`, terminated with `"\n"`, BOM-less UTF-8 via `[System.IO.File]::AppendAllText`.
- Fields: `timestamp` (ISO 8601 with offset via `DateTime.ToString('o')`), `elapsed_seconds` (numeric `[double]` straight from the existing stopwatch), `script` (literal `Invoke-AiDispatchAuto.ps1` / `Invoke-AiDispatchQueue.ps1`), `pid` (numeric `$PID`), `event` (substring before first `:`; falls back to first whitespace-delimited token; falls back to full message), `message` (exact text passed to `Write-TimingTrace`), `repo` (`{ root, slug }` read from existing `$script:RepoRoot` / `$script:repoSlug` when set).
- Trace files land under `.ai/dispatch-trace/`, which matches the existing `/.ai/dispatch-*/` ignore rule (verified via `git check-ignore`); no `.gitignore` edits.
