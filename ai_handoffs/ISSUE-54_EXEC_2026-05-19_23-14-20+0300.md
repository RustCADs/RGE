- `crates/cad-core/tests/operator_graph_snapshot_smoke.rs::operator_graph_snapshot_round_trip_preserves_nodes_and_edges`:
  New integration test file containing exactly one test. It builds an
  `OperatorGraph` with two operator nodes via public cad-core APIs —
  `OperatorNode::Cuboid(CuboidOp { width: 2.0, height: 1.0, depth: 1.5 })`
  and `OperatorNode::Transform(TransformOp { translation: [3.0, -1.5, 2.0],
  ..default })` — plus one edge from `OperatorGraph::connect(cuboid_id,
  transform_id, 0)`. It captures the inner graph with
  `GraphSnapshot::from_graph(graph.inner())`, round-trips via
  `to_ron` / `from_ron`, and materializes the deserialized snapshot with
  `to_graph()`. Assertions pin: (a) the source graph, original snapshot,
  deserialized snapshot, and restored graph each report exactly 2 nodes and
  1 edge; (b) the collected `(NodeId, OperatorNode)` and
  `(EdgeId, EdgeRecord<EdgeKind>)` vectors are equal before vs. after the
  RON round-trip; (c) the original `NodeId`s resolve in the restored graph
  to the exact expected `CuboidOp` / `TransformOp` payloads; (d) the
  original `EdgeId` resolves to an `EdgeRecord` whose `src`, `dst`, and
  `data` equal `cuboid_id`, `transform_id`, and `EdgeKind::Input(0)`,
  compared both field-by-field and as a whole `EdgeRecord`.
