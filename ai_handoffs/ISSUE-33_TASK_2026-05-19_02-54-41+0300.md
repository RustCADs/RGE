## Goal

Add the GitHub issue #33 cad-projection integration smoke proving that
`CadProjection::render_mesh_for` preserves `TransformOp`-inherited Cuboid face
labels in the same triangle order used by projection lookup. The smoke must
build a `CuboidOp -> TransformOp` graph, bind the projected entity to the
Transform root, and assert renderer-side opaque labels resolve to the same
upstream Cuboid `BRepFaceId` values as `brep_face_id_for_triangle`.

## Deliverables

- One new additive `#[test]` in
  `crates/cad-projection/tests/render_adapter_smoke.rs` covering
  `CadProjection::render_mesh_for` against a projected
  `CuboidOp -> TransformOp` root.
- The test constructs a deterministic `CuboidOp` and a non-identity
  `TransformOp`, for example a translation with default rotation and scale,
  then builds and commits a `CadGraph` where the Cuboid node feeds the
  Transform node and the graph root is the Transform node.
- The projected entity is spawned for the Transform root node, not the
  upstream Cuboid node, and its `BRepHandle.brep_owner` is set to
  `Some(ENTITY_OWNER)` before projection and lookup assertions.
- The test calls `projection.tick`, then calls both
  `projection.projected_mesh(entity)` and
  `projection.render_mesh_for(entity, &world)` on the Transform-bound entity.
- The test asserts `RenderMesh.face_labels` is `Some`, has one opaque `u64`
  label per projected triangle, and stays aligned by triangle index with the
  projected mesh's `TopologyFaceId` labels.
- For every renderer label, the test wraps the opaque `u64` as
  `TopologyFaceId(label)`, asserts it is non-degenerate for this
  Cuboid-through-Transform fixture, resolves it through the Transform-root
  graph resolver path using
  `brep_face_ids_for_node(graph.graph(), transform_node, ENTITY_OWNER)`, and
  asserts that B-Rep face ID equals the value returned by
  `projection.brep_face_id_for_triangle(entity, tri, &world, graph.graph())`.
- The test asserts the resolved IDs are the upstream Cuboid face identities
  minted under the same `ENTITY_OWNER`; it must not accept only `Some(_)`, any
  face, any owner, a sorted label set, or a deduplicated label set.
- A completed execution report packet for this dispatch.

## Acceptance Criteria

- `cargo test -p rge-cad-projection --test render_adapter_smoke` exits 0.

  and Fillet render-adapter smokes and specifically exercises
  `OperatorNode::Transform`.
- The new smoke is distinct from
  `crates/cad-projection/tests/transform_brep_face_lookup_smoke.rs`: that
  existing test covers picker-side lookup only, while this dispatch must cover
  the renderer adapter's opaque `RenderMesh.face_labels` buffer.

  successful picker lookup.

  Cuboid-through-Transform fixture because `TransformOp` should preserve
  topology and labels without adding unlabeled geometry.
- The implementation does not change renderer-tier crates, B-Rep identity
  derivation, TransformOp behavior, graph resolver behavior, projection
  lookup, picker behavior, ECS handle semantics, Cargo metadata, or unrelated
  tests.
