## Goal

Add the GitHub issue #38 cad-projection integration smoke proving that
`CadProjection::render_mesh_for` preserves face labels for a direct
`ExtrudeOp` root. The test must compare renderer-side opaque `u64` labels to
the projected `TopologyFaceId` values in exact triangle order, then prove each
renderer label resolves to the same `BRepFaceId` returned by
`brep_face_id_for_triangle` for that same triangle.

## Deliverables

- One new additive `#[test]` in
  `crates/cad-projection/tests/render_adapter_smoke.rs` covering
  `CadProjection::render_mesh_for` against a projected direct `ExtrudeOp` root
  entity.
- The test constructs a deterministic square-profile `ExtrudeOp` fixture using
  existing public APIs already imported or available to this test file. Prefer
  the existing n = 4 unit-square profile and depth 1.0 fixture used by
  `extrude_square_render_mesh_triangle_count_matches_d_projection_beta_contract`.
- The committed `CadGraph` root is the `ExtrudeOp` node itself. The projected
  entity must be spawned for that Extrude node, not for an upstream, wrapper,
  or unrelated node.
- The entity's `BRepHandle.brep_owner` is set to `Some(ENTITY_OWNER)` before
  projection and lookup assertions.
- The test calls `projection.tick`, then calls both
  `projection.projected_mesh(entity)` and
  `projection.render_mesh_for(entity, &world)` on the Extrude-bound entity.
- The test asserts `RenderMesh.face_labels` is `Some`, has exactly one opaque
  `u64` label per projected triangle, and has the same count as
  `ProjectedMesh.face_labels`.
- For every triangle index, the test asserts the renderer label equals the
  projected `TopologyFaceId.0` value at the same index. This must be an ordered
  per-triangle comparison, not a sorted, deduplicated, or set-based comparison.
- For every renderer label, the test wraps the opaque `u64` as
  `TopologyFaceId(label)`, resolves it through the Extrude-root graph resolver
  path using `brep_face_ids_for_node(graph.graph(), extrude_node,
  ENTITY_OWNER)`, and asserts that resolved `BRepFaceId` equals
  `projection.brep_face_id_for_triangle(entity, tri, &world, graph.graph())`
  for the same triangle.
- The test should pin the square-profile Extrude fixture to the existing
  D-projection-beta contract of 12 projected triangles and 12 renderer labels.
- A completed execution report packet for this dispatch.
