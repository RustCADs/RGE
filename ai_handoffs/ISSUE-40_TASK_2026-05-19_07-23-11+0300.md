## Goal

Add the GitHub issue #40 cad-projection integration smoke proving that
`CadProjection::pick_face` can pick a stable B-Rep face from a directly
projected `OperatorNode::Revolve` root. The smoke must use the existing
partial-mode square-ring Revolve fixture style, cast a deterministic ray at an
unambiguous Revolve face, and assert the returned `FacePick` identifies the
Revolve-bound entity, the expected owner, and the expected Revolve
`BRepFaceId`.

## Planner Notes

`crates/cad-projection/tests/revolve_brep_face_id_lookup_smoke.rs` already
proves triangle-index lookup for Revolve. This dispatch is different: it must
exercise the ray picker in `face_picking_smoke.rs` and prove the picked face
flows through the same entity, owner, and stable Revolve B-Rep identity that
callers consume from `FacePick`.

The intended target is the Partial-mode start cap because its expected ID is
explicit and stable:
`BRepFaceId::for_revolve_face(ENTITY_OWNER, RevolveFaceTag::StartCap { profile_count: 4 })`.
The ray `[1.25, 0.5, -5.0]` -> `[0.0, 0.0, 1.0]` hits the start cap at
`z = 0` from outside the swept half-space and avoids the square profile's fan
diagonal. The picker may see farther intersections after the cap; the
assertion is that the closest resolvable face returned by `pick_face` is the
start cap.
