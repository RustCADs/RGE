## What I Changed

### Source

- `Invoke-AiDispatchAuto.ps1`: added a pure decision helper `Get-RecoveryDecision`
  next to the other parsing helpers, and inserted a one-shot transient recovery
  block in the halt-checks section, placed strictly after the
  `.ai/dispatch.auto-halt` local sentinel and before the existing failed-auto
  `--state all` halt. The new block:
  - queries open `ai-auto` + `ai-dispatch-failed` issues with
    `--state open --json number,title,labels`;
  - delegates eligibility to `Get-RecoveryDecision`, which fails closed unless
    there is exactly one open candidate carrying exactly one
    `ai-dispatch-failure-*` taxonomy label and that label is one of the two
    explicit transient labels (`ai-dispatch-failure-stall` or
    `ai-dispatch-failure-timeout`), and not carrying
    `ai-dispatch-recovered-transient`;
  - in normal mode, idempotently ensures `ai-dispatch-recovered-transient` and
    `ai-dispatch-retry` labels exist, then performs a single `gh issue edit`
    that removes `ai-dispatch-failed` (and `ai-dispatch-done` if present) and
    adds `ai-dispatch`, `ai-dispatch-retry`, `ai-dispatch-recovered-transient`
    while keeping the original transient taxonomy label as audit evidence;
  - waits for the relabeled issue to be listable under the queue label before
    falling through to the queue-drain path so the queue does not no-op on
    GitHub's label-index lag;
  - in `-DryRun` mode prints the recovery decision and the exact intended
    label transition, then exits 0 without mutating labels and without running
    the queue;
  - on any ineligible state (zero candidates, multiple candidates, mixed
    taxonomy, non-transient taxonomy, missing taxonomy, already-recovered, or
    unknown future `ai-dispatch-failure-*` label) it falls through to the
    existing human-review halt unchanged.

## Open Questions for Reviewer

- `gh issue edit` is invoked with multiple repeated `--remove-label` and
  `--add-label` flags (one per label) rather than a single comma-joined value.
  Both forms are supported by `gh`; the repeated-flag form is preserved here
  because it matches the per-label structure of `LabelsToRemove` /
  `LabelsToAdd` returned by `Get-RecoveryDecision` and avoids quoting concerns
  on Windows. Please confirm this fits the project's `gh` invocation
  convention.
- The recovery block waits up to 60 seconds (12 × 5s) for the relabeled issue
  to appear under the queue label before falling through to the queue-drain
  step, mirroring the existing post-issue-create poll the script already uses.
  If a tighter or looser visibility budget is preferred for the recovery path,
  flag it; the value was chosen for symmetry with the existing pattern.
- An unknown future `ai-dispatch-failure-*` label (e.g.
  `ai-dispatch-failure-quota`) is treated as a non-transient taxonomy label
  and falls through to the human-review halt. This is fail-closed as the
  Task Packet requires and consistent with the verification truth table.
