## Goal

Implement GitHub issue #411 / task 156: add a narrow headless
`EditorShell` lifecycle helper that clears the shell's stored tracked CAD
entity id only when that id is stale against live CAD-world `BRepHandle`
liveness. The helper must build directly on the liveness semantics already
reported by `CadSceneInspection`, keep inspection read-only, and avoid any
render, GPU, CommandBus, save/load, graph-policy, projection-policy, camera,
selection, or UI behavior change. As the required final step, update the
autonomous task brief with exactly one gated, source/docs-read-only audit task
157 for the post-cleanup boundary; that audit must end by recording
`NEEDS_HUMAN_RECORDED` with an actual ISO date and reason, not by appending a
feature task.

## Acceptance Criteria

- `EditorShell::clear_stale_tracked_cad_entity(&mut self) -> bool` or the
  chosen local equivalent returns `true` only when `self.cad_entity` was
  previously `Some(entity)` and that entity no longer resolves to a live
  CAD-world entity carrying `BRepHandle`.
- The helper returns `false` for a fresh shell, for no tracked id, and for a
  tracked id that still resolves to a live CAD-world `BRepHandle` entity.
- The helper clears only `self.cad_entity`. It does not despawn entities, tick
  projection, restore or rollback CAD graph state, mutate selection, mutate
  camera, enqueue actions, mark dirty/saved, save/load/open/close, initialize
  render state, upload meshes, touch GPU resources, or edit forbidden surfaces.
- Render-mesh availability is not the cleanup predicate. `render_mesh_for`
  absence may remain inspection data, but stale-id cleanup is based on live
  `BRepHandle` liveness in the installed CAD world.
- Tests cover a fresh `EditorShell::new()` no-op and assert the helper returns
  `false` while inspection remains the fresh empty snapshot.
- Tests cover successful `add_cad_cuboid_to_empty_scene()` followed by helper
  no-op and assert the helper returns `false`, the tracked entity remains
  present/live, and `cad_scene_inspection()` stays consistent.
- Tests cover `restore_to(added.pre_add_head)` plus
  `despawn_brep_entity(cad_world, added.entity)` so inspection reports
  `tracked_cad_entity_present == true` and `tracked_cad_entity_live == false`
  before the helper, then the helper returns `true`, then a subsequent
  inspection reports `tracked_cad_entity_present == false`.
- Existing `cad_scene_inspection()` behavior remains read-only and continues to
  report tracked-id presence, liveness, render-mesh availability,
  frameability, selected-CAD frameability, graph shape, and B-Rep counts
  consistently before and after helper cleanup.
- `.ai/dispatch.tasks.md` appends exactly one task 157 and no task 158. Task
  157 is a GATED source/docs-read-only audit. It MUST NOT append task 158 or
  any feature/implementation task, and MUST NOT self-re-arm with a follow-up
  feature.
- Task 157's final step MUST record its conclusion as exactly one
  `NEEDS_HUMAN_RECORDED: YYYY-MM-DD - source-grounded reason` marker line
  using the actual audit date and reason, immediately followed by a
  `Recommendation for human approval` block covering proposed next feature,
  exact edit surface, risks, verification, and why that is the smallest
  coherent next step.
- Task 157 must require dispatcher-provided GitHub-state snapshot evidence for
  queue/already-filed-task claims, must compare current source, and must carry
  this same gated-audit instruction into the task 157 block it authors:
  record `NEEDS_HUMAN_RECORDED`, do not append a feature.
