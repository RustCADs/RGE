## What I Changed

### Source

- `Invoke-AiDispatchQueue.ps1`:
  - Added `[ValidateSet('', 'main', 'branch', 'pr')] [string]$PublishMode = ''` next to the existing `-NoPublish` switch and updated the `.NOTES` block to describe all three modes.
  - Added the pure helper `Resolve-DispatchPublishMode` that collapses `-PublishMode` and `-NoPublish` into one internal mode string and fails fast on an incompatible combination (e.g. `-NoPublish -PublishMode pr`).
  - Added the pure helpers `Format-DispatchPrTitle` and `Format-DispatchPrBody` that emit a deterministic PR title and body. The body includes the source issue number/title, dispatch id, branch, commit SHA, detailed log path, Codex control verdict, and the literal `Refs #<n>` linkage (never `Closes #<n>`), plus an explicit disclaimer that PR mode does not merge, push `origin/main`, or close the source issue.
  - Resolved the publish mode early in the top-level flow (`$script:ResolvedPublishMode = Resolve-DispatchPublishMode ...`) so all downstream branches key off one validated mode.
  - Extended `Format-DispatchProgressComment` to add `pr` to the `-PublishMode` ValidateSet and to render a PR-mode-specific line in the publish-decision progress comment ("pr mode - pushing the dispatch branch and opening a GitHub pull request targeting `main`; no auto-merge, no push to `origin/main`, no automatic issue close.").
  - Rewrote the publish-decision and publish blocks to branch on `$script:ResolvedPublishMode`:
    - `main` retains the existing fetch / pre-merge sync check / `git merge --ff-only` / `git push origin main` flow unchanged.
    - `pr` runs only after a branch commit exists, the loop exits 0, and Codex control verdict is `pass`. It pushes the dispatch branch with `git push --set-upstream origin <branch>`, then runs `gh pr create --repo <slug> --base main --head <branch> --title <Format-DispatchPrTitle> --body-file <Format-DispatchPrBody>`. The PR URL is parsed from `gh pr create` stdout; on a failure to parse the URL it falls back to `gh pr view <branch> --json number,url`. Any branch-push or PR-create failure is publish-pipeline failure (`publishHardFailed = $true`), so the existing taxonomy classifier emits `ai-dispatch-failure-publish`, the local branch is preserved for human recovery, and `willRetry` stays `$false`. PR mode never invokes `git merge --ff-only`, never pushes `origin/main`, and never closes the source issue.
    - `branch` keeps the legacy "kept local for human review" path.
  - Updated the result comment so the per-mode `$branchLine` and `$footerLine` describe PR mode correctly (including PR number and URL when known) and the issue-close step only fires for `main` mode, so PR mode and `branch` mode both leave the source issue open.
- `Invoke-AiDispatchAuto.ps1`:
  - Added `pr` to the `-PublishMode` ValidateSet and to the `.SYNOPSIS` / `.DESCRIPTION` / `.PARAMETER` / `.EXAMPLE` blocks.
  - Updated the queue invocation `switch` so `branch` still passes `-NoPublish` (preserving the existing test surface), `main` passes nothing (queue default), and `pr` passes `-PublishMode pr` through to the queue.
  - Updated the final summary line so it distinguishes the three modes.
- `Register-AiDispatchSchedule.ps1`:
  - Added `pr` to the `-PublishMode` ValidateSet and to the `.PARAMETER PublishMode` doc. The existing argument-string builder (`-PublishMode {0}`) is generic, so the scheduled task's `Argument` payload now naturally records `-PublishMode pr` whenever the registrar is invoked with `-Autonomous -PublishMode pr`.
- `AI_DISPATCH_AUTOMATION.md`:
  - Updated §7.1 to describe all three publish modes in the Auto layer and added a new "Queue-runner publish modes" sub-section with a three-row mode table that names the queue invocation, behavior, and the legacy `-NoPublish` equivalence for branch mode. The new section spells out the PR-mode constraints from this TASK packet verbatim (no `git merge --ff-only`, no `origin/main` push, no automatic issue close, `Refs #<n>` linkage, publish-pipeline failure classification on push or PR-create failure).
  - Extended the publish-decision progress-comment table row so it lists the new `pr` mode alongside `auto-publish`, `branch`, `not-eligible`, and `no-commit`.
