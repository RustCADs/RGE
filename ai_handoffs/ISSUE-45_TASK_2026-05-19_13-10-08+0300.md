## Deliverables

- `crates/material-graph/Cargo.toml` depends on `rge-kernel-graph-foundation = { workspace = true }` and does not add any new external dependency.
- `crates/material-graph/src/lib.rs` declares an appropriate valid failure class for this first real implementation.
- `crates/material-graph/src/lib.rs` exposes a minimal `MaterialGraph` wrapper backed by `rge_kernel_graph_foundation::Graph`, not by a local adjacency map.
- The wrapper supports adding opaque material nodes with stable substrate `NodeId` values derived deterministically from caller-provided opaque node keys or equivalent stable opaque payloads.
- The wrapper supports connecting existing nodes with edge payloads that record source and destination typed ports. The port type surface must stay data-only and minimal; do not add shader, evaluator, editor, or renderer semantics.
- Duplicate node insertion, duplicate edge insertion, and dangling endpoint insertion are surfaced as explicit errors, preserving the relevant graph-foundation failure information.

## Acceptance Criteria

- `rge-material-graph` builds and tests through `cargo test -p rge-material-graph`.
- `cargo run -q -p rge-tool-architecture-lints -- all` exits 0 after the change.
- `MaterialGraph` uses `rge_kernel_graph_foundation::{Graph, NodeId, EdgeId}` or equivalent substrate exports; it must not define local types named `NodeId`, `EdgeId`, or `StableHash`.
- The implementation does not define `BTreeMap<K, BTreeSet<K>>`, `HashMap<K, HashSet<K>>`, or equivalent local graph adjacency storage.
- Re-adding the same opaque material node in the same graph returns a duplicate-node error instead of silently succeeding.
- Re-adding the same connection with the same endpoints and same typed ports returns a duplicate-edge error instead of silently succeeding.
- Connecting from or to a node id that is not currently in the graph returns a dangling-endpoint error.
- Adding the same opaque material node key or payload in two fresh `MaterialGraph` instances produces the same `NodeId`.
- A successful connection increments edge count while preserving node count.
- The failure-class exemption count may decrease by exactly one for `crates/material-graph/Cargo.toml`; it must not increase, and no unrelated exemption may change.
- No WGSL codegen, material runtime behavior, graph editor UI, gfx dependency, shader semantics, serialization format, snapshot integration, or asset-pipeline integration is added.
