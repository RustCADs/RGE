## Goal

Add the GitHub issue #31 cad-projection integration smoke proving that
`CadProjection::render_mesh_for` preserves RoundFillet renderer-side opaque
face labels in the same triangle order used by projection lookup. The smoke
must build and project a `CuboidOp -> RoundFilletOp` root, compare inherited
non-degenerate renderer labels against `brep_face_id_for_triangle`, and prove
RoundFillet-added degenerate cap/corner labels remain present in the renderer
label buffer.

## Deliverables

- One new additive `#[test]` in
  `crates/cad-projection/tests/render_adapter_smoke.rs` covering
  `CadProjection::render_mesh_for` against a projected
  `CuboidOp -> RoundFilletOp` root.
- The test constructs a deterministic Cuboid, obtains real upstream Cuboid
  edge IDs through `cuboid.brep_edge_ids(ENTITY_OWNER)`, and constructs
  `RoundFilletOp::new(&cuboid, ENTITY_OWNER, vec![edges[0]], 0.1)`.
- The test builds and commits a `CadGraph` where the Cuboid node feeds the
  RoundFillet node, and the graph root is the RoundFillet node.
- The projected entity is spawned for the RoundFillet root node, not the
  upstream Cuboid node, and its `BRepHandle.brep_owner` is set to
  `Some(ENTITY_OWNER)` before projection and lookup assertions.
- The test calls `projection.tick`, then calls both
  `projection.projected_mesh(entity)` and
  `projection.render_mesh_for(entity, &world)` on the RoundFillet-bound
  entity.
- The test asserts `RenderMesh.face_labels` is `Some`, has one opaque `u64`
  label per projected triangle, and remains aligned by triangle index with the
  projected mesh's `TopologyFaceId` labels.
- For every inherited non-degenerate renderer label, the test wraps the opaque
  `u64` as `TopologyFaceId(label)`, resolves it through the upstream Cuboid
  face mapping for `ENTITY_OWNER`, and asserts that B-Rep face ID equals the
  value returned by
  `projection.brep_face_id_for_triangle(entity, tri, &world, graph.graph())`.
- The test asserts at least one inherited non-degenerate label is observed and
  at least one `TopologyFaceId::DEGENERATE.0` renderer label from
  RoundFillet-added cap/corner geometry remains present as an opaque label.
- A completed execution report packet for this dispatch.

## Acceptance Criteria

- `cargo test -p rge-cad-projection --test render_adapter_smoke` exits 0.
- The new smoke is distinct from the existing Cuboid and Extrude render-adapter
  smokes and specifically exercises `OperatorNode::RoundFillet`.
- No `BRepEdgeId` is synthesized by hand; every edge passed into
  `RoundFilletOp::new` comes from the upstream Cuboid's `BRepEdgeProvider`
  surface.
- The entity under test is bound to the RoundFillet root, so the assertion
  proves renderer-label alignment through the projected RoundFillet output
  rather than through an upstream Cuboid projection.
- The renderer-label assertion compares exact upstream Cuboid `BRepFaceId`
  values for inherited non-degenerate labels. The test must not accept only
  `Some(_)`, any face, any owner, or a sorted/deduplicated label set.
- Degenerate RoundFillet labels are not silently filtered out by the render
  adapter: the renderer-side `RenderMesh.face_labels` vector contains at least
  one `TopologyFaceId::DEGENERATE.0` entry.
- The implementation does not change production B-Rep identity derivation,
  RoundFillet tessellation, projection lookup, picker behavior, ECS handles,
  Cargo metadata, or unrelated tests.
- If no real render-adapter bug is exposed, tracked project-file changes are

## Planner Notes

Use the existing `render_mesh_face_labels_resolve_consistently_with_picker`
test in `render_adapter_smoke.rs` as the closest renderer-label alignment
model, but add a separate RoundFillet-specific smoke instead of replacing or
generalizing the Cuboid coverage. Use
`crates/cad-projection/tests/round_fillet_brep_face_id_lookup_smoke.rs` only as
read-only reference for constructing the `CuboidOp -> RoundFilletOp` graph,
spawning the entity on the RoundFillet root, and identifying the inherited
non-degenerate versus `TopologyFaceId::DEGENERATE` label cases.

The intended core assertion is: for each RoundFillet output triangle, the
renderer-side opaque label is the same `TopologyFaceId.0` value carried by the
projected mesh. If that label is non-degenerate, resolving it through the
upstream Cuboid face mapping for `ENTITY_OWNER` must produce the same
`BRepFaceId` as `CadProjection::brep_face_id_for_triangle` for the same
triangle. If that label is `TopologyFaceId::DEGENERATE.0`, the label remains in
`RenderMesh.face_labels` as an opaque renderer label rather than being filtered
out by the adapter.
