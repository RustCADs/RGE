### Source

- `crates/material-graph/Cargo.toml`: added `rge-kernel-graph-foundation = { workspace = true }`
  under `[dependencies]` (no new external dependency). Corrected the package
  `description` from `"material graph IR + WGSL codegen"` to
  `"material graph foundation: typed-port node/edge wrapper over the graph substrate"`,
  since WGSL codegen is explicitly out of scope for this foundation slice.
- `crates/material-graph/src/lib.rs`: replaced the stub with the minimal
  foundation wrapper. Module-level doc now declares `//! Failure class:
  snapshot-recoverable` (matches `cad-core`'s sibling `Graph`-backed wrapper —
  rebuildable structural state that participates in snapshot/restore). New
  public surface:
  - `MaterialGraphError(pub GraphError)` — thin newtype preserving the exact
    graph-foundation failure; implements `Display`/`Error`/`From<GraphError>`.
    `Clone` is intentionally omitted because the substrate `GraphError` is not
    `Clone`.
  - `PortType` — data-only `#[repr(u8)]` tag (`Scalar`/`Vector`/`Color`/`Texture`);
    no shader/evaluator/renderer semantics, no type-compatibility validation.
  - `MaterialEdge { src_port, dst_port }` — data-only edge payload recording
    the typed source and destination ports.
  - `MaterialGraph` — wraps `rge_kernel_graph_foundation::Graph<MaterialNode,
    MaterialEdge>` (no local adjacency storage; no local `NodeId`/`EdgeId`/
    `StableHash` types). Methods: `new`/`default`, `add_node(&str) ->
    Result<NodeId, _>` (substrate `NodeId` derived deterministically via
    `NodeId::from_bytes` over the opaque key), `connect(NodeId, NodeId,
    MaterialEdge) -> Result<EdgeId, _>` (`EdgeId` derived from both endpoints
    plus both port tags, so identical connections collide while differing
    ports stay distinct), `node_count`, `edge_count`.
  - Private `MaterialNode { key: String }` node payload and private
    `material_edge_id` id-derivation helper.
  Duplicate node, duplicate edge, and dangling endpoint are surfaced directly
  from the substrate `Graph` mutators via `MaterialGraphError`.
- `tools/architecture-lints/exemptions.toml`: removed only the
  `failure-class` exemption entry for `crates/material-graph/Cargo.toml`
  (the new valid `//! Failure class: snapshot-recoverable` declaration makes
  the lint pass). No other exemption changed; no exemption added or broadened.

### Tests

Inline `#[cfg(test)] mod tests` in `crates/material-graph/src/lib.rs`:

- `node_ids_are_stable_across_graphs`: same key (`"albedo"`) in two fresh
  `MaterialGraph` instances yields the same `NodeId`.
- `distinct_keys_get_distinct_ids`: keys `"a"` / `"b"` yield distinct `NodeId`s.
- `connect_succeeds_and_updates_counts`: a successful `connect` raises edge
  count `0 -> 1` while node count stays `2`.
- `duplicate_node_is_rejected`: re-adding key `"a"` returns
  `GraphError::DuplicateNode`.
- `duplicate_edge_is_rejected`: re-adding an identical connection returns
  `GraphError::DuplicateEdge` and leaves edge count at `1`.
- `differing_ports_are_not_duplicates`: two connections between the same
  endpoints with different `PortType` pairs are distinct edges (count `2`).
- `dangling_endpoint_is_rejected`: connecting to a never-added node returns
  `GraphError::DanglingEndpoint` and leaves edge count at `0`.
- `empty_graph_has_zero_counts`: `MaterialGraph::default()` reports `0`/`0`.
