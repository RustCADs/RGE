## Deliverables

- Extend `crates/editor-egui-host/src/menu.rs` command-palette filtering in the existing helper path. Do not introduce a second command model or a separate palette command registry.
- Match each whitespace-separated filter term against the already-projected menu-path label, shortcut display, and `Command::diagnostic_id()` fields. Each term must still match at least one field.
- Preserve current case-insensitive exact word/field, prefix, and substring matches. These match classes must sort ahead of fuzzy ordered-subsequence matches for the same filter.
- Add deterministic fuzzy ordered-subsequence matching as the next match class after substring. A fuzzy term match means all query characters appear in order within one field after the same normalization used by the current filter.
- Replace or revise the current scalar scoring so the ordering is explainable as a stable tuple. The tuple must expose match class first, then stable tie-breakers such as fuzzy gap count, matched field priority, field length, label length, and original menu order.
- Keep blank-filter behavior unchanged: a blank or whitespace-only filter returns all projected entries in original menu order.
- Keep disabled entries visible in filtered results and keep selection/activation behavior on the existing helpers.
- Add focused `crates/editor-egui-host/src/menu_tests.rs` coverage for fuzzy-only matches, exact/prefix/substring outranking fuzzy-only matches, stable ordering for multiple fuzzy matches, and no-match behavior for queries that cannot match even fuzzily.
- Update `plans/BASELINE.md`, `Status.md`, `HANDOFF.md`, `change.md`, and task 98 in `.ai/dispatch.tasks.md` to record the shipped fuzzy behavior and remaining open non-goals.
- Produce this dispatch's EXEC packet with the implementation summary, tests/gates run, any local verification blockers, and final git status.

## Acceptance Criteria

- `filter_command_palette_entries()` still operates over `ProjectedCommandPaletteEntry` rows produced by the existing menu projection.
- No new command model, command registry, plugin action execution path, host-shell FIFO replacement, keybinding editor, or editor-shell routing change is added.
- The filter remains an AND over whitespace-separated terms: every term must match at least one of label, shortcut display, or diagnostic id.
- Match classes are ordered as exact word/field before prefix before substring before fuzzy ordered-subsequence before no match.
- A row that matches a filter without using fuzzy matching must sort ahead of a row that only matches the same filter through fuzzy ordered-subsequence matching.
- Fuzzy tie-breaking is deterministic and documented in code/tests through an inspectable score tuple or equivalent named type; original menu order is the final tie-breaker.
- Fuzzy matching is case-insensitive and uses the same ASCII lowercasing style as the current filter unless a local helper already provides a safer equivalent.
- The helper does not allocate or persist command history and does not modify palette selection state except through the existing filtered result ordering.
- Tests include at least one fuzzy-only query that the current exact/prefix/substring-only helper would not match.
- Tests include an ordering case where exact, prefix, or substring matches outrank fuzzy-only matches for the same filter.
- Tests include stable ordering for at least two fuzzy-only matches.
- Tests include a no-match query that returns an empty list even after fuzzy matching is available.
- Existing tests for blank filters, shortcut search, diagnostic-id search, multi-term matching, exact-word ordering, keyboard selection, selected-row focus, and disabled-row behavior remain passing or are updated only to reflect the intentional fuzzy scoring behavior.
- `plans/BASELINE.md` gets a forward-only command-palette fuzzy-scoring subsection above older command-palette history; older historical sections are not rewritten except where the new current-state entry must supersede open fuzzy wording.
- `Status.md`, `HANDOFF.md`, and `change.md` describe the new current fuzzy behavior and preserve remaining open non-goals.
- `.ai/dispatch.tasks.md` task 98 is marked done or otherwise updated using the repo's existing forward-only task-list style after the behavior ships.
