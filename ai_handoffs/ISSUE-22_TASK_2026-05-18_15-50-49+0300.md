## Planner Notes

Reference commit `6807cb6` is useful for the `TransformOp` behavior only. The
important semantic decision is that Transform is topology-preserving: it moves
positions, leaves triangle topology unchanged, and therefore must pass face
labels through one-for-one. The stale tests should be reconciled to that
contract rather than preserving issue #18's old failing verification state.

The topology lineage smoke was reported during issue #18 verification, but it
does not itself define Transform behavior. Treat it as a required targeted
rerun and diagnosis point. Only change it if the rerun on this branch proves
that the new label-preserving contract intentionally changes the lineage
expectation.
