### Tests

- `crates/cad-projection/tests/render_adapter_smoke.rs::transform_render_mesh_face_labels_align_with_projection_lookup`:
  one new additive `#[test]` (GitHub issue #33). It builds a deterministic
  1×1×1 `CuboidOp` and a non-identity `TransformOp` (translation
  `[3.0, -2.0, 1.0]`, default rotation + unit scale), commits a `CadGraph`
  where the Cuboid node feeds the Transform node and the graph root is the
  Transform node, then spawns/binds the projected entity to the **Transform
  root** node (not the upstream Cuboid) with `BRepHandle.brep_owner` set to
  `Some(ENTITY_OWNER)`. After `projection.tick`, it queries both
  `projected_mesh(entity)` and `render_mesh_for(entity, &world)` and pins:
  - `RenderMesh.face_labels` is `Some`, with one opaque `u64` label per
    projected triangle, equal in length to the projected mesh's
    `TopologyFaceId` label buffer.
  - Renderer label N exactly equals the projected `TopologyFaceId.0` at the
    same triangle index N — proving order and value preservation, not just
    `Some(_)`.
  - Every renderer label is non-degenerate (`assert_ne!` against
    `TopologyFaceId::DEGENERATE`) — TransformOp is topology-preserving and
    adds no nameless cap/corner geometry.
  - Each label, wrapped as `TopologyFaceId(label)`, resolves through the
    Transform-root graph resolver
    `brep_face_ids_for_node(graph.graph(), transform_node, ENTITY_OWNER)` to a
    `BRepFaceId` that equals (a) the upstream Cuboid face identity minted via
    `cuboid.brep_face_ids(ENTITY_OWNER)` for the same `TopologyFaceId`, and
    (b) the value returned by
    `brep_face_id_for_triangle(entity, tri, &world, graph.graph())` for the
    same triangle. The assertion compares exact upstream Cuboid `BRepFaceId`
    values; it does not accept only `Some(_)`, any face, any owner, a sorted
    label set, or a deduplicated label set.
- File-level module doc: added an issue-#33 bullet to the smoke summary and a
