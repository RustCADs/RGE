## What I Changed

### Source

- `crates/cad-core/src/operators/sweep.rs`: imported `TopologyFaceId`
  alongside `Tessellation` (`use crate::tessellation::{Tessellation,
  TopologyFaceId};`).
- `crates/cad-core/src/operators/sweep.rs` (`SweepOp::evaluate`): after
  the existing positions/indices construction (unchanged), build a
  per-triangle `face_labels: Vec<TopologyFaceId>` in the canonical
  emission order — first cap `n - 2` triangles all `TopologyFaceId(0)`,
  last cap `n - 2` triangles all `TopologyFaceId(1)`, then side quads in
  segment-major / edge-major order with each side quad's two triangles
  labeled `TopologyFaceId(2 + (k * n + i))` for path segment `k` and
  profile edge `i`. Successful output now returns
  `Tessellation::with_labels(positions, indices, face_labels)` instead of
  `Tessellation::new(positions, indices)`. Geometry generation,
  validation, winding correction, vertex/triangle order, and
  `structural_hash` are untouched — the label loops run in parallel with
  the existing index emission and reuse the existing `cap_tris` /
  `side_tris` counts for capacity.
- `crates/cad-core/src/operators/sweep.rs` (`impl Operator for SweepOp`):
  added an `output_is_labeled` override returning `true` unconditionally,
  matching `evaluate`'s now-always-labeled output (mirrors `ExtrudeOp`),
  so the cache-key contract `output_is_labeled == evaluate(...).is_labeled()`
  holds.

### Tests

- `crates/cad-core/src/operators/sweep.rs::tests::sweep_output_is_labeled_returns_true`:
  renamed/updated from the prior `sweep_output_is_labeled_returns_false`
  (which asserted `!output_is_labeled`); now asserts `output_is_labeled(&[])`
  is `true`, consistent with the labeled-output contract.
- `crates/cad-core/src/operators/sweep.rs::tests::sweep_emits_canonical_face_labels_for_2_point_path`:

  triangle_count() == 12`, first two triangles `TopologyFaceId(0)`, next
  two `TopologyFaceId(1)`, then four side quads (2 triangles each)
  `TopologyFaceId(2..=5)`.
- `crates/cad-core/src/operators/sweep.rs::tests::sweep_emits_canonical_face_labels_for_multi_segment_path`:
  triangle profile (`n = 3`) along a 3-point path (`m = 3`, two
  segments). Pins `labels.len() == 14`, caps `0` and `1`, and that side
  labels advance segment-before-edge — segment 0 → faces `2, 3, 4`,
  segment 1 → faces `5, 6, 7`, two triangles per side quad.
