## Goal

Add the GitHub issue #34 cad-projection integration smoke proving that
`CadProjection::render_mesh_for` preserves `SweepOp` face labels in the same
triangle order used by `ProjectedMesh.face_labels` and picker-side B-Rep face
lookup. The smoke must project a Sweep-root entity with
`BRepHandle.brep_owner` set, compare renderer labels to projected labels by
triangle index, and prove every non-degenerate renderer label resolves to the
same `BRepFaceId` as `brep_face_id_for_triangle` for that exact triangle.

## Deliverables

- One new additive `#[test]` in
  `crates/cad-projection/tests/render_adapter_smoke.rs` covering
  `CadProjection::render_mesh_for` against a projected Sweep-root entity.
- The test constructs a deterministic `SweepOp`, preferably the existing
  square-profile, 3-point monotonic-Z fixture pattern from
  `sweep_brep_face_id_lookup_smoke.rs`, and commits a `CadGraph` whose root is
  the Sweep node.
- The projected entity is spawned for the Sweep root node, and its
  `BRepHandle.brep_owner` is set to `Some(ENTITY_OWNER)` before projection and
  lookup assertions.
- The test calls `projection.tick`, then calls both
  `projection.projected_mesh(entity)` and
  `projection.render_mesh_for(entity, &world)` on the Sweep-bound entity.
- The test asserts `RenderMesh.face_labels` is `Some`, has one opaque `u64`
  label per projected triangle, and stays aligned by triangle index with the
  projected mesh's `TopologyFaceId` labels.
- For every non-degenerate renderer label, the test wraps the opaque `u64` as
  `TopologyFaceId(label)`, resolves it through the Sweep-root graph resolver
  path using `brep_face_ids_for_node(graph.graph(), sweep_node, ENTITY_OWNER)`,
  and asserts that B-Rep face ID equals the value returned by
  `projection.brep_face_id_for_triangle(entity, tri, &world, graph.graph())`
  for the same triangle.
- The test should assert the square-profile, 3-point monotonic-Z Sweep fixture
  produces the expected 20 projected triangles and 20 renderer labels unless an
  existing helper already pins that count in the same test.
- The test must not accept only `Some(_)`, any face, any owner, a sorted label
  set, or a deduplicated label set.
- A completed execution report packet for this dispatch.

## Acceptance Criteria

- The new smoke is distinct from
  `crates/cad-projection/tests/sweep_brep_face_id_lookup_smoke.rs`: that
