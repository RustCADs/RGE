## What I Changed

### Source

- `crates/material-graph/src/lib.rs`: converted the working-copy file's line
  terminators from CRLF to LF (`perl -i -pe 's/\r\n/\n/g'`). No Rust source
  text, tests, docs, declarations, imports, ordering, or whitespace other than
  the newline encoding was touched. Behavior is unchanged.

  **Root cause.** `git config core.autocrlf` is `true` on this Windows machine.
  The repository blob for `crates/material-graph/src/lib.rs` is stored with LF,
  but `autocrlf` rewrote it to CRLF in the working copy on checkout.
  `rustfmt.toml` sets `newline_style = "Unix"`, so the canonical fmt gate
  (`cargo +nightly fmt --all -- --check`, run by `.ai/dispatch.verify.ps1`)
  flagged the CRLF terminators in that file. The anim-graph files edited by the
  original execution round were authored through the editor tooling and written
  with LF, so they passed; only the freshly-checked-out, untouched MaterialGraph
  file carried CRLF and failed the gate.

  Because the repository blob is already LF (and `autocrlf` re-normalizes
  CRLF->LF on the way into the index), this normalization produces **no
  committable diff**: `git diff -- crates/material-graph/src/lib.rs` and
  `git diff --numstat -- crates/material-graph/src/lib.rs` both report no
  content change. The change is purely a working-tree byte normalization so the
  on-disk file rustfmt reads matches the LF the repository already stores.
