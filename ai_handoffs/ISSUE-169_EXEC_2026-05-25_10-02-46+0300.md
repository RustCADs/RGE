## What I Changed

### Source

- `kernel/ecs/src/archetype.rs`:
  - `Archetype::has_component_at_row` changed from `pub fn` to `pub(crate) fn`
    (kernel/ecs/src/archetype.rs:171). The method's body, signature, and
    semantics are otherwise untouched; its only out-of-file caller —
    `EntityRef::contains` in `kernel/ecs/src/entity.rs` — already lives inside
    the `rge-kernel-ecs` crate, so the crate-private visibility is sufficient.
    The in-file `tests` module also still consumes it (intra-crate).
  - The doc comment on the public `has_component` accessor used to point users
    at `[`Archetype::has_component_at_row`]` for the row-specific check; that
    link would now reach a `pub(crate)` item from a `pub` doc comment and risk
    a `private_intra_doc_links` lint. The link is now
    `[`EntityRef::contains`](crate::entity::EntityRef::contains)`, which is the
    actual public surface that exposes row-specific membership. No behavior
    change.
- No other source files were touched.

### Tests

- `kernel/ecs/tests/sparse_rows.rs::heterogeneous_snapshot_round_trip_exercises_erased_nonzero_row_insert`:
  rewritten to make the "Speed first restores at row 2" property
  deterministic. Now:
  1. Spawns three entities into an array `ids`,
  2. Calls `ids.sort()` to put them in ascending `EntityId` order (which is
     exactly the order the snapshot serializer emits and the restore consumes
     — `World::serialize_snapshot` sorts by `e.ulid().0` ascending, and
     `World::restore_from_snapshot` walks that wire order),
  3. Destructures into `let [first, middle, last] = ids;` and places `Speed`
     only on `last`. Restore therefore re-spawns `first` at row 0, `middle` at
     row 1, `last` at row 2, and `Speed`'s first ever erased insert during
     restore lands on row 2 (>0) without any reliance on `Ulid::new`'s
     intra-millisecond ordering. All downstream assertions were rewritten to
     key off `first`/`middle`/`last` instead of `a`/`b`/`c`, preserving the
     original presence/absence and byte-identical-reserialize invariants.
  The other 10 tests in `sparse_rows.rs` were unchanged.
