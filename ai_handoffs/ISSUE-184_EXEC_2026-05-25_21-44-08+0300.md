## What I Changed

### Source

- `Invoke-AiDispatchLoop.ps1`:
  - Param block: added top-level `[switch]$EnablePreflightAudit` (default off,
    no other parameter sets touched).
  - Added function `Invoke-CodexPreflightAudit`: runs `Invoke-CodexPrompt`
    with `-Sandbox read-only`, captures stdout to
    `.ai/dispatch-<id>/codex.preflight.log`, extracts a marker-delimited
    Markdown body (`<<<PREFLIGHT_AUDIT_BEGIN>>>` / `<<<PREFLIGHT_AUDIT_END>>>`,
    last-occurrence anchored), validates the four required headings
    (`# Pre-flight Audit`, `## Boundary`, `## Pitfall Checklist`,
    `## Verification Checklist`), parses `- [ ] Pn:` and `- [ ] Vn:` items
    from the right sections only, requires at least one of each, and fails
    on duplicate Pn or Vn IDs. Writes
    `.ai/dispatch-<id>/codex.preflight.md` only after every validation step
    succeeds; on any failure `Fail` is invoked and no file is written, so a
    stale pre-existing checklist cannot leak into a later step.
  - Updated `Invoke-ClaudeExecute`: added `[string]$PreflightChecklist = ''`
    parameter; when non-empty AND `Round -eq 0`, prepends an "Advisory
    Codex pre-flight checklist (round 0 only)" block to the execute prompt,
    followed by an explicit boundary statement that the TASK packet remains
    authoritative, the checklist does not expand scope or authorize
    out-of-envelope edits/tests, and conflicting items must be ignored. The
    existing `EXEC_STATUS:` / `EXEC_PACKET:` tail marker contract and
    Markers hashtable are unchanged.
  - Updated `Invoke-CodexControl`: added `[string]$PreflightChecklist = ''`
    parameter; when non-empty, injects the same advisory block plus the
    boundary statement on every control round, placed between the
    "Latest execution report:" header and the "Also inspect:" list. The
    Codex JSON output schema (`.ai/codex_control.schema.json`) and
    `--output-schema` / `--output-last-message` flow are unchanged.
  - Main script body: after TASK finalization / resume location and after
    the `-PlanOnly` exit, when `$EnablePreflightAudit` is set, calls
    `Invoke-CodexPreflightAudit -TaskPacket $taskPacket` exactly once
    before the execute loop. Stores the validated checklist in a local
    `$preflightChecklist` variable that defaults to `''` so omitted-switch
    invocations stay byte-identical to prior behavior, and threads that
    variable into every `Invoke-ClaudeExecute` and `Invoke-CodexControl`
    call inside the loop.

`Invoke-CodexPrompt`, `Invoke-WithTimeout`, the stall watchdog,
