Implement GitHub issue #391 / task 137: in `editor-shell`, handle the winit
cursor-leave lifecycle event by clearing stale cursor state, resetting the
viewport left-double-click tracker, cancelling any active viewport right-button
orbit and middle-button pan drags, and releasing `CursorGrabMode::None` only
when a viewport drag was active before the cursor left. This is the
cursor-leave sibling of ISSUE-389's focus-loss drag cancellation. Preserve
camera math, viewport hit testing, picking, wheel zoom, frame-all, command
routing, shortcuts, plugin/runtime, OS clipboard, CAD/CommandBus, save/load,

## Acceptance Criteria

- `WindowEvent::CursorLeft` clears `self.cursor_pos`; after leave,
  `WindowEvent::CursorMoved` remains the only path that installs a fresh
  cursor position.
- Cursor leave resets the viewport left-double-click tracker so a left press
  after re-entry cannot pair with a stale pre-leave press to frame the scene.
- If an orbit drag is active before cursor leave, cursor leave stops the orbit
  drag, releases the drag cursor grab once, and later cursor movement no longer
  applies orbit deltas until a new drag starts.
- If a pan drag is active before cursor leave, cursor leave stops the pan drag,
  releases the drag cursor grab once, and later cursor movement no longer
  applies pan deltas until a new drag starts.
- If both orbit and pan drags are active before cursor leave, cursor leave
  stops both states and emits exactly one `CursorGrabMode::None` release after
  both states are idle.
- If no viewport drag is active before cursor leave, cursor leave still clears
  cursor position and resets the double-click tracker, but emits no cursor-grab
  release request.
- Existing focus-loss behavior from ISSUE-389 remains unchanged, including
  active orbit/pan cancellation, combined-drag release-once behavior, no-active
  no-release behavior, and focus-gain preservation.
- Existing camera math, viewport hit testing, picking, wheel zoom, frame-all
  behavior except stale double-click reset on leave, command routing,
  shortcuts, plugin/runtime, OS clipboard, CAD/CommandBus, undo/dirty, and
  save/load behavior are unchanged.
- `.ai/dispatch.tasks.md`, `Status.md`, `HANDOFF.md`, `plans/BASELINE.md`, and
  `change.md` record task 137 as done; task 138 is not appended.
- `git diff --name-only` shows only MAY-edit files plus generated ISSUE-391
  artifacts, and `git diff --check` is clean.
