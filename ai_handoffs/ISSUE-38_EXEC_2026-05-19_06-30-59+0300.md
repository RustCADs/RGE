- `crates/cad-projection/tests/render_adapter_smoke.rs::extrude_render_mesh_face_labels_align_with_projection_lookup`:
  new additive `#[test]` (Test 10) for GitHub issue #38. It builds a
  deterministic graph whose committed root is a direct `ExtrudeOp` node
  (the existing n=4 unit-square profile `[[0,0],[1,0],[1,1],[0,1]]`,
  depth `1.0` fixture mirrored from
  `extrude_square_render_mesh_triangle_count_matches_d_projection_beta_contract`).
  It spawns the projected entity for the `ExtrudeOp` node itself, sets
  `BRepHandle.brep_owner = Some(ENTITY_OWNER)` before projection, ticks,
  then calls both `projection.projected_mesh(entity)` and
  `projection.render_mesh_for(entity, &world)`. It pins the
  D-projection-β contract of 12 projected triangles and 12 renderer
  labels, asserts `RenderMesh.face_labels` is `Some` with one opaque
  `u64` per projected triangle and equal length to
  `ProjectedMesh.face_labels`, then performs an ordered per-triangle
  comparison that each renderer `u64` equals the projected
  `TopologyFaceId.0` at the same index (no sort/dedup/set comparison).
  For every renderer label it wraps the `u64` as `TopologyFaceId(label)`,
  resolves it through
  `brep_face_ids_for_node(graph.graph(), extrude_node, ENTITY_OWNER)`,
  asserts the label is non-degenerate, and asserts the resolved
  `BRepFaceId` equals
  `projection.brep_face_id_for_triangle(entity, tri, &world, graph.graph())`
  for the same triangle.
