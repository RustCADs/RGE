## Audit findings

### Sharpest current abstraction pain point

**The `editor-actions::Action` trait's
`apply(&self, world: &mut World)` and `revert(&self, world: &mut World)`
World-only signatures cannot mediate CAD-store (`CadGraph` +
`CadProjection`) mutations, blocking the natural next class of
user-driven editor dispatches.** PLAN §6.16.3's two-layer Command Bus
model with a `BusEntry::CadCheckpoint { id, description }` variant is
specified but unimplemented; the `BusEntry` enum currently ships only the
`Action(Box<dyn Action>)` variant with `#[non_exhaustive]` marking the
future slot.

Concrete file:line evidence (re-read this dispatch):

- `crates/editor-actions/src/action.rs:87`:
  `fn apply(&self, world: &mut World) -> Result<(), ActionResult>;` — the
  trait method every editor mutation must implement; binds the Action to
  `rge_kernel_ecs::World` only.
- `crates/editor-actions/src/action.rs:96`:
  `fn revert(&self, world: &mut World) -> Result<(), ActionResult>;` —
