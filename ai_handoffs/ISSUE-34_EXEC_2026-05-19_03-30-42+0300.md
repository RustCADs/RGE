### Tests

- `crates/cad-projection/tests/render_adapter_smoke.rs::sweep_render_mesh_face_labels_align_with_projection_lookup`:
  new additive integration smoke for GitHub issue #34. It builds a
  `SweepOp`-root `CadGraph` from the canonical square-profile (n=4, CCW),
  3-point monotonic-Z path (m=3) fixture mirrored read-only from
  `sweep_brep_face_id_lookup_smoke.rs`, spawns the projected entity for the
  Sweep root node, sets `BRepHandle.brep_owner = Some(ENTITY_OWNER)`
  post-spawn, and ticks the projection. It then:
  * asserts the projected mesh has exactly 20 triangles
    (`2*n*(m-1) + 2*(n-2) = 20`);
  * asserts `RenderMesh.face_labels` is `Some` with exactly 20 opaque `u64`
    labels, equal to both the projected triangle count and the projected
    `face_labels` length;
  * asserts renderer label N equals the projected `TopologyFaceId.0` at the
    same triangle index N (order- and value-preservation, not just `Some(_)`);
  * asserts no `TopologyFaceId::DEGENERATE` label appears (SweepOp mints
    stable cap/side labels);
  * for every renderer label, resolves it through
    `brep_face_ids_for_node(graph.graph(), sweep_node, ENTITY_OWNER)` and
    asserts the resulting `BRepFaceId` equals
    `brep_face_id_for_triangle(entity, tri, &world, graph.graph())` for the
    exact same triangle.
  The assertion does not accept `Some(_)`, any face, any owner, a sorted
  label set, or a deduplicated label set.
