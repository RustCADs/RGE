## Goal

Add the GitHub issue #35 cad-projection integration smoke proving that
`CadProjection::render_mesh_for` preserves `LoftOp` face labels in the same
triangle order used by `ProjectedMesh.face_labels` and picker-side B-Rep face
lookup. The smoke must build and project a simple Loft-root entity, call
`render_mesh_for`, compare renderer labels to projected labels by triangle
index, and prove each renderer label resolves to the same `BRepFaceId` as
`brep_face_id_for_triangle` for that exact triangle.

## Deliverables

- One new additive `#[test]` in
  `crates/cad-projection/tests/render_adapter_smoke.rs` covering
  `CadProjection::render_mesh_for` against a projected Loft-root entity.
- The test constructs a deterministic `LoftOp`, preferably the existing
  square-profile to larger-square profile fixture pattern from
  `loft_brep_face_id_lookup_smoke.rs`, and commits a `CadGraph` whose root is
  the Loft node.
- The projected entity is spawned for the Loft root node, and its
  `BRepHandle.brep_owner` is set to `Some(ENTITY_OWNER)` before projection and
  lookup assertions.
- The test calls `projection.tick`, then calls both
  `projection.projected_mesh(entity)` and
  `projection.render_mesh_for(entity, &world)` on the Loft-bound entity.
- The test asserts `RenderMesh.face_labels` is `Some`, has one opaque `u64`
  label per projected triangle, and stays aligned by triangle index with the
  projected mesh's `TopologyFaceId` labels.
- For every renderer label, the test wraps the opaque `u64` as
  `TopologyFaceId(label)`, resolves it through the Loft-root graph resolver
  path using `brep_face_ids_for_node(graph.graph(), loft_node, ENTITY_OWNER)`,
  and asserts that B-Rep face ID equals the value returned by
  `projection.brep_face_id_for_triangle(entity, tri, &world, graph.graph())`
  for the same triangle.
- The test should assert the square-profile Loft fixture produces the expected
  12 projected triangles and 12 renderer labels unless an existing helper
  already pins that count in the same test.
- The test must not accept only `Some(_)`, any face, any owner, a sorted label
  set, or a deduplicated label set.
- `crates/cad-projection/tests/render_adapter_smoke.rs` must have the newline
  style accepted by repository rustfmt. The prior attempt failed because
  `cargo +nightly fmt --all -- --check` reported incorrect newline style in
  this file.
- A completed execution report packet for this dispatch.
